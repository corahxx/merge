# 动态区域统计字段可行性分析

## 📋 需求概述

根据查询的颗粒度动态调整区域统计字段：
- **全局查询**（无区域筛选）→ 按 `省份_中文` 统计
- **省份级别查询**（选择了省份）→ 按 `城市_中文` 统计
- **城市级别查询**（选择了城市）→ 按 `区县_中文` 统计

---

## ✅ 技术可行性分析

### 1. 数据字段确认

**✅ 完全可行**

- 数据库表 `evdata` 中已存在以下字段：
  - `省份_中文` (VARCHAR(255))
  - `城市_中文` (VARCHAR(255))
  - `区县_中文` (TEXT)

- 所有字段都可用于统计和分组
- 字段值完整，适合进行 value_counts() 统计

### 2. 查询颗粒度判断

**✅ 完全可行**

- 当前已有 `region_filter` 参数传递查询条件
- `region_filter` 格式：`{'province': 'XX省', 'city': 'XX市', 'district': 'XX区'}`
- 可以根据 `region_filter` 的内容判断查询颗粒度：
  - 无 `region_filter` 或为空 → 全局查询
  - 有 `province` 但无 `city` → 省份级别查询
  - 有 `city` 但无 `district` → 城市级别查询
  - 有 `district` → 城市级别查询（已是最细粒度，仍按区县统计）

### 3. 动态统计字段实现

**✅ 完全可行**

- 在 `DataAnalyzer.get_statistics()` 中根据 `region_filter` 动态选择统计字段
- 可以同时统计多个级别，或根据颗粒度选择单一字段
- 使用 pandas 的 `value_counts()` 方法，支持任意字段统计

### 4. 向后兼容性

**✅ 完全可行**

- 保留现有的 `by_location` 字段（按区县统计）
- 新增 `by_province`、`by_city` 字段
- 在显示层根据查询颗粒度选择使用哪个字段

**优点**：
- 逻辑清晰，易于理解

**缺点**：
- 需要修改多个地方的代码
- 需要处理字段名的变化

**核心逻辑**：

**缺点**：
- 性能稍差（需要统计三个字段）
- 数据量较大（特别是区县级别）

## ⚠️ 潜在问题与解决方案

### 1. 字段名不一致问题

**问题**：
- 不同级别的统计使用不同的 key（`by_province`、`by_city`、`by_location`）
- 显示层需要判断使用哪个 key

**解决方案**：
- 统一使用 `by_location` 作为 key，但在统计时根据颗粒度选择不同的字段
- 或者在 stats 中添加 `location_level` 字段标识统计级别

### 2. 向后兼容性问题

**问题**：
- 现有代码可能依赖 `by_location` 字段
- 如果改为 `by_province` 或 `by_city`，可能影响其他功能

**解决方案**：
- 同时保留多个字段（`by_province`、`by_city`、`by_location`）
- 在显示层根据颗粒度选择使用哪个字段
- 或者统一使用 `by_location`，但根据颗粒度统计不同字段

### 3. 数据量问题

**问题**：
- 省份数量：约 34 个（全国）
- 城市数量：约 300+ 个
- 区县数量：约 3000+ 个

**解决方案**：
- 根据颗粒度选择统计字段，避免统计过多数据
- 限制显示数量（如：前 20 个）

### 4. 查询颗粒度判断逻辑

**问题**：
- 需要准确判断用户的查询颗粒度
- 边界情况处理（如：只选择了省份，但想看到区县统计）

**解决方案**：
- 明确判断逻辑：
  - 无 `region_filter` → 省份统计
  - 有 `province` 但无 `city` → 城市统计
  - 有 `city` → 区县统计
- 如果用户需要更细粒度的统计，可以调整筛选条件

```python
def determine_location_field(region_filter: Optional[Dict[str, str]]) -> Tuple[str, str]:
    """
    根据 region_filter 确定统计字段和 key
    :param region_filter: 区域筛选条件
    :return: (统计字段名, stats key)
    """
    if not region_filter or not region_filter.get('province'):

    elif not region_filter.get('city'):
```

### 2. 显示层字段选择逻辑

```python
def get_location_stat_key(region_filter: Optional[Dict[str, str]], stats: Dict) -> Optional[str]:
    """
    根据 region_filter 和 stats 确定使用的统计字段 key
    :param region_filter: 区域筛选条件
    :param stats: 统计数据
    :return: 统计字段 key
    """
    if not region_filter or not region_filter.get('province'):
        # 全局查询 → 使用省份统计
        return 'by_province' if 'by_province' in stats.get('basic_stats', {}) else None
    elif not region_filter.get('city'):
        # 省份级别查询 → 使用城市统计
        return 'by_city' if 'by_city' in stats.get('basic_stats', {}) else None
    else:
        # 城市级别查询 → 使用区县统计
        return 'by_location' if 'by_location' in stats.get('basic_stats', {}) else None
```
