# 区域三级联动查询功能可行性分析

## 📋 需求概述

在数据管理系统中增加按区域查询的选项，实现三级联动菜单（省份→城市→区县），从数据库读取 `省份_中文`、`城市_中文`、`区县_中文` 三个字段，支持数据缓存和手动更新功能。

---

## ✅ 技术可行性分析

### 1. 数据库字段确认

**✅ 完全可行**

- 数据库表 `evdata` 中已存在以下字段：
  - `省份_中文` (VARCHAR(255), nullable)
  - `城市_中文` (VARCHAR(255), nullable)
  - `区县_中文` (TEXT, nullable)

- 字段类型适合进行字符串查询和筛选
- 字段允许 NULL 值，需要在查询时处理空值情况

### 2. 三级联动菜单实现

**✅ 完全可行**

- **Streamlit 支持**：`st.selectbox` 支持动态更新选项
- **联动逻辑**：
  - 第一级：选择省份 → 触发城市列表更新
  - 第二级：选择城市 → 触发区县列表更新
  - 第三级：选择区县 → 完成筛选条件

- **技术方案**：
  ```python
  # 使用 st.selectbox 和 session_state 实现联动
  selected_province = st.selectbox("省份", provinces)
  cities = get_cities_by_province(selected_province)  # 根据省份筛选城市
  selected_city = st.selectbox("城市", cities)
  districts = get_districts_by_city(selected_province, selected_city)  # 根据城市筛选区县
  selected_district = st.selectbox("区县", districts)
  ```

### 3. 数据字典类设计

**✅ 完全可行**

- **类设计思路**：
  - 创建 `RegionDictionary` 类，负责区域数据的加载、缓存、更新
  - 使用 `@st.cache_data` 或 `session_state` 实现缓存
  - 提供手动刷新方法

- **类结构设计**：
  ```python
  class RegionDictionary:
      - load_from_database()  # 从数据库加载数据
      - get_provinces()       # 获取省份列表
      - get_cities(province)  # 根据省份获取城市列表
      - get_districts(province, city)  # 根据省份和城市获取区县列表
      - refresh_cache()       # 手动刷新缓存
      - is_cached()          # 检查是否已缓存
  ```

### 4. 缓存机制

**✅ 完全可行**

- **Streamlit 缓存方案**：
  - 使用 `st.session_state` 存储区域数据字典
  - 首次加载时从数据库读取，后续使用缓存
  - 提供"更新区域列表"按钮，手动触发重新加载

- **性能考虑**：
  - 区域数据量相对较小（全国约3000+区县）
  - 一次查询即可获取所有区域数据
  - 缓存后查询速度极快（内存操作）

### 5. 查询条件集成

**✅ 完全可行**

- **现有支持**：
  - `DataAnalyzer.get_statistics()` 已支持 `location_filter` 参数
  - 当前实现使用 `区县_中文` 字段进行筛选

- **扩展方案**：
  - 支持按省份、城市、区县三个级别筛选
  - 可以组合使用（如：选择省份+城市，或只选择省份）
  - SQL 条件构建：
    ```sql
    WHERE `省份_中文` = 'XX省' 
      AND `城市_中文` = 'XX市' 
      AND `区县_中文` = 'XX区'
    ```

## ⚠️ 潜在问题与解决方案

### 1. 数据质量问题

**问题**：
- 数据库中可能存在 NULL 值
- 区域名称可能不统一（如："北京市" vs "北京"）
- 可能存在数据重复或错误

**解决方案**：
- 查询时过滤 NULL 值：`WHERE 省份_中文 IS NOT NULL`
- 使用 `DISTINCT` 去重
- 对区域名称进行标准化处理（可选）

### 2. 性能问题

**问题**：
- 如果数据量很大，首次加载可能较慢
- 三级联动需要频繁查询数据

**解决方案**：
- 使用缓存机制，只加载一次
- 使用 `st.session_state` 存储数据，避免重复查询
- 限制查询范围（如：只查询有数据的区域）

### 3. 数据更新问题

**问题**：
- 数据库新增区域数据后，缓存不会自动更新
- 用户需要手动刷新才能看到新数据

**解决方案**：
- 提供明显的"更新区域列表"按钮
- 在按钮旁边显示最后更新时间
- 可选：设置自动刷新间隔（如：每天自动刷新一次）

### 4. 空值处理

**问题**：
- 某些记录可能只有省份，没有城市或区县
- 某些记录可能只有省份和城市，没有区县

**解决方案**：
- 在构建联动菜单时，只显示有完整数据的区域
- 或者在查询时支持部分匹配（如：只选择省份，不选择城市和区县）

### 5. 数据量问题

**问题**：
- 全国有3000+区县，下拉框选项可能过多
- 用户体验可能受影响

**解决方案**：
- 使用搜索功能（Streamlit 的 `st.selectbox` 支持搜索）
- 或者使用 `st.multiselect` 支持多选（但不符合三级联动需求）
- 限制显示数量，使用分页（不推荐，影响体验）
