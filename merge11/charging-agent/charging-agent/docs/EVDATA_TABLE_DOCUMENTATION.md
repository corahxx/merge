## 数据库与表概述

### 数据库信息
- **数据库名称**: `evcipadata`
- **字符集**: `utf8mb4`
- **排序规则**: `utf8mb4_unicode_ci`

### 表信息
- **表名**: `evdata`
- **表类型**: 宽表（扁平化设计）
- **主要用途**: 存储充电桩及其关联的充电站、运营商、地区等完整信息
- **数据特点**: 每条记录代表一个充电桩，包含该充电桩的所有相关信息

## 程序常用字段重点说明

### 🌟 核心标识字段

#### 1. **UID** (唯一标识ID)
- **类型**: `VARCHAR(255)`
- **可空**: ❌ **不可为空**
- **说明**: 使用UUID字符串作为每条记录的唯一标识
- **用途**:

  - 数据关联
- **注意事项**: 该字段是唯一且不可为空的，是数据记录的唯一标识

#### 2. **充电桩编号**
- **类型**: `VARCHAR(255)`
- **可空**: ✅
- **说明**: 充电桩的业务编号
- **用途**: 
  - 业务层面的充电桩标识
  - 数据比对时的关键字段
  - 用户查询的主要依据

### 📍 区域相关字段（高频使用）

#### 4. **城市_中文**
- **类型**: `VARCHAR(255)`
- **可空**: ✅
- **说明**: 城市的中文名称（如：北京市、上海市、广州市）
- **程序使用**:
  - 用于区域筛选：`WHERE 城市_中文 = 'XX市'`
  - 用于区域统计：`GROUP BY 城市_中文`
  - 用于三级区域查询的城市级别筛选
- **示例值**: `"北京市"`, `"上海市"`, `"广州市"`, `"济南市"`

#### 5. **区县_中文**
- **类型**: `TEXT`
- **可空**: ✅
- **说明**: 区县的中文名称（如：朝阳区、海淀区）

- **可空**: ✅

  - 用于功率筛选，支持以下功率区间：

    - `60-120kW（大功率）`: `额定功率 > 60 AND 额定功率 <= 120`
    - `>120kW（超快充）`: `额定功率 > 120`
  - 用于功率统计和分布分析
  - 用于生成功率分布图表
- **注意事项**: 查询时需要处理 `NULL` 值

## 数据关系说明

### 数据层级关系

```
省份_中文
  └── 城市_中文
      └── 区县_中文
          └── 所属充电站编号
              └── 充电桩编号 (UID)
```

### 数据冗余说明

由于采用宽表设计，`evdata` 表存在以下数据冗余：

1. **充电站信息冗余**: 同一充电站的所有充电桩记录都包含相同的充电站信息（充电站名称、位置、投入使用时间等）
2. **运营商信息冗余**: 同一运营商的所有充电桩记录都包含相同的运营商信息
3. **区域信息冗余**: 同一区域的所有充电桩记录都包含相同的区域信息

**影响**:
- ✅ 优点：查询简单，无需JOIN操作
- ⚠️ 缺点：存储空间较大，数据更新需要同步多条记录

## 使用建议

### 1. 查询优化建议

#### 区域查询
```sql
-- ✅ 推荐：使用中文字段进行查询
SELECT * FROM evdata WHERE 省份_中文 = '北京市' AND 城市_中文 = '北京市';

-- ❌ 不推荐：使用数字编码字段（需要额外转换）
SELECT * FROM evdata WHERE 省份 = 110000;
```

#### 运营商查询
```sql
-- ✅ 推荐：使用标准化后的运营商名称
SELECT * FROM evdata WHERE 运营商名称 IN ('国家电网', '特来电');

-- ❌ 不推荐：使用原始字段（可能包含多种变体）
SELECT * FROM evdata WHERE 充电桩所属运营商 LIKE '%国家电网%';
```

#### 类型查询
```sql
-- ✅ 推荐：使用转换后的类型字段
SELECT * FROM evdata WHERE 充电桩类型_转换 LIKE '%直流%';

-- ❌ 不推荐：使用原始类型字段（可能格式不统一）
SELECT * FROM evdata WHERE 充电桩类型 = 'DC';
```

#### 功率查询
```sql
-- ✅ 推荐：明确处理NULL值
SELECT * FROM evdata 
WHERE 额定功率 > 60 AND 额定功率 <= 120 
  AND 额定功率 IS NOT NULL;

-- ❌ 不推荐：忽略NULL值处理
SELECT * FROM evdata WHERE 额定功率 > 60 AND 额定功率 <= 120;
```

#### 时间查询
```sql
-- ✅ 推荐：对于VARCHAR类型的时间字段，使用STR_TO_DATE转换
SELECT * FROM evdata 
WHERE STR_TO_DATE(充电站投入使用时间, '%Y/%m/%d') >= '2025-01-01'
  AND 充电站投入使用时间 IS NOT NULL 
  AND 充电站投入使用时间 != '';

-- ✅ 推荐：对于DATE类型的时间字段，直接比较
SELECT * FROM evdata 
WHERE 充电桩生产日期 >= '2025-01-01';
```

### 2. 统计查询建议

#### 运营商统计（TOP10）
```sql
SELECT 运营商名称, COUNT(*) as 数量
FROM evdata
WHERE 运营商名称 IS NOT NULL AND 运营商名称 != ''
GROUP BY 运营商名称
ORDER BY 数量 DESC
LIMIT 10;
```

#### 区域统计（根据查询级别动态选择）
```sql
-- 全局查询：按省份统计
SELECT 省份_中文, COUNT(*) as 数量
FROM evdata
WHERE 省份_中文 IS NOT NULL
GROUP BY 省份_中文
ORDER BY 数量 DESC
LIMIT 10;
```

#### 功率统计
```sql
SELECT 
  CASE 
    WHEN 额定功率 <= 7 THEN '≤7kW（慢充）'
    WHEN 额定功率 > 7 AND 额定功率 <= 30 THEN '7-30kW（小功率）'
    WHEN 额定功率 > 30 AND 额定功率 <= 60 THEN '30-60kW（中功率）'
    WHEN 额定功率 > 60 AND 额定功率 <= 120 THEN '60-120kW（大功率）'
    WHEN 额定功率 > 120 THEN '>120kW（超快充）'
    ELSE '未知'
  END as 功率区间,
  COUNT(*) as 数量
FROM evdata
WHERE 额定功率 IS NOT NULL
GROUP BY 功率区间;
```

### 3. 数据质量注意事项

1. **NULL值处理**: 大部分字段允许为NULL，查询时需要适当处理
2. **空字符串处理**: 某些字段可能为空字符串而非NULL，需要同时检查：`IS NOT NULL AND != ''`
3. **数据标准化**: 优先使用带 `_转换` 后缀的标准化字段
4. **运营商名称**: 空值会自动填充为 `"无登记运营商"`，但查询时仍需检查
5. **时间字段格式**: `充电站投入使用时间` 是VARCHAR类型，需要格式转换；`充电桩生产日期` 和 `设备开通时间` 是DATE类型，可直接比较

### 4. 性能优化建议

1. **索引建议**: 
   - 在 `UID` 字段上建立主键索引（如果尚未建立）
   - 在 `运营商名称`、`省份_中文`、`城市_中文`、`区县_中文` 上建立索引
   - 在 `所属充电站编号` 上建立索引
   - 在 `充电桩类型_转换`、`额定功率` 上建立索引（如果经常用于筛选）

2. **查询优化**:
   - 使用 `LIMIT` 限制查询结果数量
   - 避免使用 `SELECT *`，只选择需要的字段
   - 对于大表查询，考虑使用分区或分页

3. **数据更新**:
   - 由于数据冗余，更新充电站信息时需要同步更新所有相关充电桩记录
   - 建议使用事务确保数据一致性

### 常用查询模板

#### 综合筛选查询
```sql
SELECT 
  充电桩编号,
  运营商名称,
  省份_中文,
  城市_中文,
  区县_中文,
  充电桩类型_转换,
  额定功率,
  所属充电站编号,
  充电站名称
FROM evdata
WHERE 运营商名称 IN ('国家电网', '特来电')
  AND 省份_中文 = '山东省'
  AND 城市_中文 = '济南市'
  AND 充电桩类型_转换 LIKE '%直流%'
  AND 额定功率 > 60
  AND 额定功率 IS NOT NULL
  AND STR_TO_DATE(充电站投入使用时间, '%Y/%m/%d') >= '2025-01-01'
  AND 充电站投入使用时间 IS NOT NULL
  AND 充电站投入使用时间 != '';
```

#### 统计汇总查询
```sql
SELECT 
  运营商名称,
  充电桩类型_转换,
  COUNT(*) as 充电桩数量,
  COUNT(DISTINCT 所属充电站编号) as 充电站数量,
  AVG(额定功率) as 平均功率,
  MIN(额定功率) as 最小功率,
  MAX(额定功率) as 最大功率
FROM evdata
WHERE 运营商名称 IS NOT NULL
  AND 充电桩类型_转换 IS NOT NULL
  AND 额定功率 IS NOT NULL
GROUP BY 运营商名称, 充电桩类型_转换
ORDER BY 充电桩数量 DESC;
```
