# 数据类型转换指南

## 功能概述

系统现在支持根据数据库字段类型自动转换EXCEL数据格式，确保数据格式与数据库字段类型匹配，解决入库行数为0的问题。

## 工作原理

### 1. 自动检测数据库表结构

系统会自动获取目标数据表的结构信息，包括：
- 字段名称
- 字段类型（VARCHAR, INT, DATETIME等）
- 是否允许空值（nullable）

### 2. 自动类型转换

根据数据库字段类型，系统会自动转换EXCEL数据：

| 数据库类型 | EXCEL数据类型 | 转换规则 |
|-----------|--------------|---------|
| **VARCHAR/CHAR/TEXT** | 任意 | 转换为字符串，清理'nan'、'None'等无效值 |
| **INT/INTEGER/BIGINT** | 字符串数字、浮点数 | 转换为整数（使用Int64可空类型） |
| **FLOAT/DOUBLE/DECIMAL** | 字符串数字、整数 | 转换为浮点数 |
| **DATE** | 字符串日期 | 转换为日期类型 |
| **DATETIME/TIMESTAMP** | 字符串日期时间 | 转换为日期时间类型 |
| **BOOLEAN/BOOL** | 字符串/数字 | 识别true/false、1/0、是/否等，转换为布尔值 |

### 3. 转换流程

```
EXCEL数据读取
    ↓
字段映射（如果有）
    ↓
去除空白
    ↓
标准化字段值（运营商、区域等）
    ↓
【新增】根据数据库字段类型转换数据类型 ← 这里！
    ↓
处理空值
    ↓
去重
    ↓
数据验证
    ↓
插入数据库
```

## 使用说明

### 自动启用

类型转换功能已自动集成到数据处理流程中，无需额外配置。系统会：

1. **自动获取表结构** - 在初始化时读取目标表的字段类型
2. **自动转换数据类型** - 在数据清洗阶段进行类型转换
3. **显示转换日志** - 在verbose模式下显示转换过程

### 查看转换信息

在verbose模式下，你会看到类似这样的输出：

```
✅ 字段 '充电桩编号' 已转换为 INT
✅ 字段 '运营商名称' 已转换为 VARCHAR
✅ 字段 '充电开始时间' 已转换为 DATETIME
类型转换字段: 3 个
```

## 常见问题

### Q1: 为什么入库行数还是0？

**可能原因：**
1. 数据类型转换失败 - 查看日志中的转换错误
2. 数据验证失败 - 关键字段为空导致整行被删除
3. 数据库约束冲突 - 唯一键冲突、外键约束等

**解决方案：**
1. 查看详细日志，检查转换错误
2. 检查数据质量报告
3. 尝试使用更宽松的数据验证规则

### Q2: 如何知道哪些字段被转换了？

查看处理日志中的"类型转换字段"统计信息。

### Q3: 可以禁用类型转换吗？

目前类型转换是自动启用的，确保数据格式正确。如果遇到问题，可以：
1. 检查数据质量
2. 查看转换错误日志
3. 手动调整EXCEL数据格式

### Q4: 转换失败的数据会怎样？

- 如果某个字段转换失败，会记录错误但继续处理
- 转换失败的数据会保持原始格式
- 如果数据库插入时仍然失败，该行会被跳过

## 类型转换示例

### 示例1：整数类型转换

**数据库字段：** `充电桩编号 INT`

**EXCEL数据：** 
- "123" (字符串) → 转换为 123 (整数)
- "123.5" (浮点数) → 转换为 123 (整数)
- "" (空字符串) → 转换为 NULL

### 示例2：日期时间转换

**数据库字段：** `充电开始时间 DATETIME`

**EXCEL数据：**
- "2024-01-15 10:30:00" → 转换为 datetime对象
- "2024/1/15" → 转换为 datetime对象
- "" (空字符串) → 转换为 NULL

### 示例3：字符串清理

**数据库字段：** `运营商名称 VARCHAR(100)`

**EXCEL数据：**
- "星星充电" → "星星充电"
- "nan" → "" (空字符串，MySQL会转换为NULL)
- "  特来电  " → "特来电" (已去除空白)

## 最佳实践

1. **检查表结构** - 确保了解目标表的字段类型
2. **数据预处理** - 在EXCEL中尽量保持数据格式一致
3. **查看日志** - 关注类型转换的警告和错误
4. **验证结果** - 导入后检查数据是否正确

## 技术细节

- 使用pandas的`to_numeric()`, `to_datetime()`等进行类型转换
- 使用`errors='coerce'`参数，转换失败时返回NaN而不是抛出异常
- 整数类型使用`Int64`（可空整数类型）以正确处理NULL值
- 字符串类型统一转换为string类型
