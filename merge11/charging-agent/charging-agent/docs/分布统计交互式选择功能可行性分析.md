## 2. 技术可行性分析

### 2.1 Plotly 饼图能力

**结论：✅ 完全支持**

Plotly 饼图本身是**数据驱动**的，传入不同的 DataFrame 即可显示不同内容。当前代码使用 `px.pie()` 创建饼图：

```python
fig = px.pie(
    df_clean,
    values=values_col,
    names=names_col,
    title=title
)
```

只需根据用户选择过滤 `df_clean`，饼图即可动态更新。

### 2.2 Streamlit 交互组件

**可选方案：**

| 组件 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| `st.multiselect` | 下拉多选 | 简洁、节省空间 | 不能同时显示数量 |
| `st.checkbox` 循环 | 逐行复选 | 可显示详细信息 | 代码较繁琐 |
| `st.data_editor` | 可编辑表格 | 功能强大 | 复杂度高 |

**推荐方案：`st.multiselect` + 数据表格联动**

```python
# 1. 获取全部运营商数据并排序
operator_df = pd.DataFrame(
    list(bs['by_operator'].items()),
    columns=['运营商', '数量']
).sort_values('数量', ascending=False)

# 2. 默认选中前10
default_selection = operator_df['运营商'].head(10).tolist()

# 3. 多选组件
selected_operators = st.multiselect(
    "选择要显示的运营商",
    options=operator_df['运营商'].tolist(),
    default=default_selection,
    key="operator_select"
)

# 4. 根据选择过滤数据
filtered_df = operator_df[operator_df['运营商'].isin(selected_operators)]

# 5. 生成饼图
fig = px.pie(filtered_df, values='数量', names='运营商', title='运营商分布')
```

### 2.3 状态管理

使用 `st.session_state` 管理选择状态，避免页面刷新时重置：

```python
# 初始化状态
if 'operator_selection' not in st.session_state:
    st.session_state.operator_selection = default_top10

# 读取/更新状态
selected = st.multiselect(..., default=st.session_state.operator_selection)
```

## 3. 实现方案

### 3.1 方案一：下拉多选 + 表格展示（推荐）

**布局设计：**

```
┌─────────────────────────────────┬───────────────────────────────┐
│                                 │  选择运营商: [多选下拉框]      │
│         🍩 动态饼图              │                               │
│                                 │  [快捷按钮: 前10 | 前20 | 全部] │
│                                 │  ─────────────────────────     │
│                                 │  运营商    │ 数量    │ 占比   │
│                                 │  ─────────────────────────     │
│                                 │  星星充电   │ 709,998 │ 23.6% │
│                                 │  云快充     │ 672,948 │ 22.4% │
│                                 │  ...       │ ...     │ ...   │
└─────────────────────────────────┴───────────────────────────────┘
```

**优点：**
- 界面简洁
- 下拉框支持搜索
- 表格可显示完整数据

**缺点：**
- 多选框与表格分离，不够直观

### 3.2 方案二：可编辑表格 + 复选框列

**布局设计：**

```
┌─────────────────────────────────┬───────────────────────────────┐
│                                 │  [全选] [前10] [清空]          │
│         🍩 动态饼图              │  ─────────────────────────     │
│                                 │  ☑ │ 运营商    │ 数量    │占比 │
│                                 │  ☑ │ 星星充电   │ 709,998│23.6%│
│                                 │  ☑ │ 云快充     │ 672,948│22.4%│
│                                 │  ☐ │ 快电       │  85,234│ 2.8%│
│                                 │  ...                          │
└─────────────────────────────────┴───────────────────────────────┘
```

**实现方式：**

```python
# 使用 st.data_editor 创建可编辑表格
operator_df['选中'] = operator_df['运营商'].isin(default_top10)

edited_df = st.data_editor(
    operator_df,
    column_config={
        "选中": st.column_config.CheckboxColumn(
            "选中",
            help="勾选以加入饼图",
            default=False,
        )
    },
    disabled=["运营商", "数量"],
    hide_index=True,
)

# 根据选中状态过滤
selected_df = edited_df[edited_df['选中'] == True]
```

**优点：**
- 复选框与数据直接关联，操作直观
- 可同时查看所有数据

**缺点：**
- `st.data_editor` 性能开销较大
- 数据量大时可能卡顿

### 4.2 核心代码修改

**修改后代码：**

```python
# 运营商统计饼图（交互式）
if 'by_operator' in bs and bs['by_operator']:
    st.subheader("📊 运营商分布统计")
    
    # 1. 获取全部数据并排序
    operator_df = pd.DataFrame(
        list(bs['by_operator'].items()),
        columns=['运营商', '数量']
    ).sort_values('数量', ascending=False).reset_index(drop=True)
    
    # 2. 计算占比
    total = operator_df['数量'].sum()
    operator_df['占比'] = (operator_df['数量'] / total * 100).round(2).astype(str) + '%'
    
    col_operator1, col_operator2 = st.columns([1, 1])
    
    with col_operator2:
        # 3. 快捷选择按钮
        st.markdown("**📌 快捷选择**")
        btn_cols = st.columns(4)
        with btn_cols[0]:
            if st.button("前10", key="op_top10"):
                st.session_state.selected_operators = operator_df['运营商'].head(10).tolist()
        with btn_cols[1]:
            if st.button("前20", key="op_top20"):
                st.session_state.selected_operators = operator_df['运营商'].head(20).tolist()
        with btn_cols[2]:
            if st.button("前50", key="op_top50"):
                st.session_state.selected_operators = operator_df['运营商'].head(50).tolist()
        with btn_cols[3]:
            if st.button("全部", key="op_all"):
                st.session_state.selected_operators = operator_df['运营商'].tolist()
        
        # 4. 初始化默认选择
        if 'selected_operators' not in st.session_state:
            st.session_state.selected_operators = operator_df['运营商'].head(10).tolist()
        
        # 5. 多选下拉框
        selected_operators = st.multiselect(
            "🔍 自定义选择运营商",
            options=operator_df['运营商'].tolist(),
            default=st.session_state.selected_operators,
            key="operator_multiselect"
        )
        
        # 6. 显示全部运营商表格（带高亮）
        st.markdown(f"**📊 运营商列表** (已选 {len(selected_operators)} / {len(operator_df)})")
        display_df = operator_df.copy()
        display_df['已选'] = display_df['运营商'].isin(selected_operators)
        st.dataframe(
            display_df[['运营商', '数量', '占比']],
            use_container_width=True,
            hide_index=True
        )
    
    with col_operator1:
        # 7. 根据选择生成饼图
        if selected_operators:
            filtered_df = operator_df[operator_df['运营商'].isin(selected_operators)]
            fig_operator_pie = handlers['analysis'].create_pie_chart(
                filtered_df,
                '数量',
                '运营商',
                f'运营商分布饼图 ({len(selected_operators)}个)'
            )
            if fig_operator_pie:
                st.plotly_chart(fig_operator_pie, use_container_width=True)
        else:
            st.info("⚠️ 请至少选择一个运营商")
```

### 4.3 区域分布同理

区域分布的修改逻辑与运营商完全一致，仅需替换字段名：
- `'运营商'` → `'区域'`
- `'by_operator'` → `location_key`（动态，如 `by_province`、`by_city` 等）

## 5. 性能考量

### 5.1 数据量预估

| 统计类型 | 预估数据量 | 性能影响 |
|----------|------------|----------|
| 运营商 | 50-200个 | 低 |
| 省份 | 34个 | 极低 |
| 城市 | 300-400个 | 低 |
| 区县 | 3000+个 | 中等 |

### 5.2 优化措施

1. **虚拟滚动表格**：区县数据量大时，使用 `st.dataframe` 自带虚拟滚动
2. **默认限制**：区县默认显示前50，避免列表过长
3. **延迟加载**：仅在用户点击"查看全部"时加载完整列表
