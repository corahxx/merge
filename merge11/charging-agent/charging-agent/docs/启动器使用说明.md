# 充电桩数据管理系统 - 启动器使用说明

> **版本**: 2.0  
> **更新**: 2026-01-14

## 📋 概述

轻量级启动器（`charging-agent-launcher.exe`）是一个独立的可执行文件，用于：
- 配置数据库连接信息
- 配置LLM大模型开关
- 测试数据库连接
- 启动Streamlit应用
- 自动打开浏览器

**特点**：
- ✅ 打包启动器和Python环境（约120MB）
- ✅ 源代码以文件夹形式存在
- ✅ 代码修改后无需重新打包，重启即可生效
- ✅ 集成账号管理和权限控制

---

## 🚀 快速开始

### 1. 打包启动器

#### 方式1：使用批处理脚本（推荐）

```bash
# Windows
build_launcher.bat
```

#### 方式2：手动打包

```bash
# 1. 安装PyInstaller（如果未安装）
pip install pyinstaller

# 2. 执行打包
pyinstaller launcher.spec
```

打包完成后，EXE文件位于：`dist/charging-agent-launcher.exe`

### 2. 部署结构

将打包后的EXE文件复制到项目根目录，确保目录结构如下：

```
charging-agent/
├── charging-agent-launcher.exe  (启动器，约120MB)
│
├── main.py                      (主入口-带登录)
├── app.py                       (AI助手页面)
├── data_manager.py              (数据管理页面)
├── pile_model_query.py          (充电桩查询)
├── agent.py                     (AI智能体)
├── config.py                    (配置文件)
│
├── auth/                        (认证模块) ⭐必需
├── services/                    (服务层) ⭐必需
├── pages/                       (管理页面) ⭐必需
├── core/                        (AI核心) ⭐必需
├── data/                        (数据处理) ⭐必需
├── handlers/                    (业务处理) ⭐必需
├── utils/                       (工具函数) ⭐必需
│
├── assets/                      (静态资源-LOGO)
├── .streamlit/                  (Streamlit配置-主题)
│   └── config.toml
│
└── uploads/                     (上传文件目录)
```

**注意**：
- 如果源代码在项目根目录（不在`app/`子目录），启动器会自动检测并使用根目录
- 配置文件`config.py`会在源代码目录中生成

### 3. 首次使用

1. **双击启动器**：`charging-agent-launcher.exe`
2. **配置数据库**：
   - 如果`config.py`不存在，会自动弹出配置界面
   - 输入数据库信息（主机、端口、用户、密码、数据库名）
   - 点击"测试连接并保存"
3. **启动应用**：
   - 连接成功后，自动启动Streamlit
   - 自动打开浏览器（http://localhost:8501）

### 4. 后续使用

1. **双击启动器**：`charging-agent-launcher.exe`
2. **自动加载配置**：从`config.py`读取配置
3. **自动测试连接**：验证数据库连接
4. **启动应用**：如果连接成功，自动启动

## 🔧 配置说明

### 配置文件位置

配置文件`config.py`位于源代码目录中：
- 如果源代码在`app/`目录：`app/config.py`
- 如果源代码在根目录：`config.py`

### 配置文件格式

```python
# config.py - 数据库配置
# 支持环境变量覆盖，优先级：环境变量 > 配置文件

import os

DB_CONFIG = {
    'host': os.getenv('DB_HOST', 'localhost'),
    'port': int(os.getenv('DB_PORT', '3306')),
    'user': os.getenv('DB_USER', 'root'),
    'password': os.getenv('DB_PASSWORD', 'your_password'),
    'database': os.getenv('DB_NAME', 'evcipadata')
}
```

### 修改配置

**方式1：使用启动器配置界面**
- 启动器会自动检测配置
- 如果连接失败，会提示重新配置

**方式2：手动编辑config.py**
- 直接编辑配置文件
- 重启启动器即可生效

## 🐛 故障排除

### 问题1：启动器无法找到源代码

**症状**：提示"未找到应用入口文件"

**解决方案**：
1. 确保源代码目录存在
2. 检查目录结构是否正确
3. 如果源代码在根目录，确保`main.py`或`data_manager.py`存在

### 问题2：数据库连接失败

**症状**：提示"数据库连接失败"

**解决方案**：
1. 检查数据库服务是否启动
2. 检查配置信息是否正确
3. 检查网络连接
4. 使用启动器重新配置

### 问题3：Streamlit启动失败

**症状**：启动器启动但浏览器未打开

**解决方案**：
1. 检查端口8501是否被占用
2. 手动访问：http://localhost:8501
3. 查看启动器控制台输出的错误信息

### 问题4：导入模块失败

**症状**：提示"ModuleNotFoundError"

**解决方案**：
1. 确保所有依赖已安装（在开发环境）
2. 检查源代码目录结构
3. 确保`__init__.py`文件存在

## 📦 打包说明

### 打包内容

启动器EXE包含：
- ✅ Python解释器
- ✅ Streamlit框架
- ✅ 基础依赖包（pandas, sqlalchemy等）
- ✅ 启动器代码

**不包含**：
- ❌ 源代码（以文件夹形式存在）
- ❌ 用户配置（运行时生成）
- ❌ 上传文件（运行时创建）

### 文件大小

- **启动器EXE**: 50-100MB
- **源代码目录**: 通常<10MB
- **总部署大小**: 60-110MB

### 优化建议

1. **排除不需要的模块**：在`launcher.spec`中配置`excludes`
2. **使用UPX压缩**：已启用（`upx=True`）
3. **分离可选功能**：如AI助手功能（LangChain）可以排除

## 🔄 更新流程

### 更新源代码

1. 直接修改源代码文件
2. 重启启动器
3. 无需重新打包

### 更新启动器

1. 修改`launcher.py`
2. 重新打包：`build_launcher.bat`
3. 替换旧的EXE文件

### 更新依赖

如果源代码需要新的Python包：

1. **开发环境**：在虚拟环境中安装
   ```bash
   pip install new_package
   ```

2. **打包环境**：如果新包是启动器需要的，需要重新打包
   - 在`launcher.spec`的`hiddenimports`中添加
   - 重新打包
