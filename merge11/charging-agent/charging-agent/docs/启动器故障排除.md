# 启动器故障排除指南

## 🔍 问题：网页无法打开

如果启动器显示"应用已启动"，但浏览器无法打开网页，请按以下步骤排查：

### 步骤1：检查控制台输出

查看启动器控制台是否有错误信息：
- ❌ Streamlit进程已退出
- ❌ 连接失败
- ⚠️ 等待超时

### 步骤2：使用调试模式

使用调试模式可以看到Streamlit的实时输出：

```bash
# 如果使用Python脚本
python launcher.py --debug

# 如果使用EXE（需要重新打包时添加参数支持）
# 或直接修改launcher.py中的debug_mode = True
```

### 步骤3：手动测试Streamlit

运行诊断脚本：

```bash
python test_streamlit_manual.py
```

这个脚本会：
- 检查文件是否存在
- 检查依赖是否安装
- 测试数据库连接
- 直接启动Streamlit（可以看到完整输出）

### 步骤4：检查常见问题

#### 问题1：端口被占用

**症状**：启动器提示"端口8501已被占用"

**解决方案**：
1. 关闭其他Streamlit实例
2. 或修改启动器使用其他端口

#### 问题2：依赖缺失

**症状**：Streamlit启动失败，显示`ModuleNotFoundError`

**解决方案**：
```bash
pip install -r requirements.txt
```

#### 问题3：代码错误

**症状**：Streamlit启动但立即退出，显示Python错误

**解决方案**：
1. 使用调试模式查看完整错误
2. 检查源代码是否有语法错误
3. 检查导入的模块是否存在

#### 问题4：数据库连接失败

**症状**：应用启动但无法访问数据库

**解决方案**：
1. 检查数据库服务是否运行
2. 检查配置文件是否正确
3. 使用启动器重新配置

### 步骤5：查看详细日志

如果问题仍然存在，可以：

1. **启用调试模式**（修改launcher.py）：
   ```python
   debug_mode = True  # 在start_streamlit函数调用前
   ```

2. **手动运行Streamlit**：
   ```bash
   streamlit run main.py --server.headless=true
   ```

3. **检查Streamlit日志**：
   - Windows: `%USERPROFILE%\.streamlit\logs\`
   - 查看最新的日志文件

### 步骤6：常见错误信息

#### "ModuleNotFoundError: No module named 'xxx'"
- **原因**：依赖未安装
- **解决**：`pip install xxx` 或 `pip install -r requirements.txt`

#### "Address already in use"
- **原因**：端口被占用
- **解决**：关闭占用端口的进程，或使用其他端口

#### "Connection refused"
- **原因**：Streamlit服务未启动或已崩溃
- **解决**：使用调试模式查看详细错误

#### "数据库连接失败"
- **原因**：数据库配置错误或服务未运行
- **解决**：检查配置和数据库服务

## 🛠️ 诊断工具

### 工具1：test_streamlit_manual.py

手动测试脚本，用于诊断问题：

```bash
python test_streamlit_manual.py
```

### 工具2：检查端口

```bash
# Windows
netstat -ano | findstr :8501

# 查看进程
tasklist | findstr <PID>
```

### 工具3：检查Python环境

```bash
python --version
python -c "import streamlit; print(streamlit.__version__)"
```
