## 二、问题根因分析

### 2.1 异常数据特征

通过对 90,227 条异常记录的分析，发现以下特征：

| 分析维度 | 结果 | 条数 | 占比 |
|----------|------|------|------|
| **城市代码 = 区县代码** | 完全相同 | 86,699 | **96.1%** |
| 城市前4位 = 区县前4位 | 部分匹配 | 887 | 1.0% |
| 无匹配关系 | 不匹配 | 2,641 | 2.9% |

### 2.2 根因判定

**主要原因（96.1%）**：数据源在"城市"字段中**错误填入了区县代码**

示例：
```
错误数据：城市 = 310115（浦东新区的区县代码）
正确数据：城市 = 310100（上海市的城市代码）
```

### 2.3 省份代码一致性验证

| 检查项 | 结果 | 条数 |
|--------|------|------|
| 城市代码前2位 = 省份代码前2位 | **一致** | 59,508 |
| 城市代码前2位 ≠ 省份代码前2位 | 不一致 | 176 |
| 省份代码为空 | N/A | 30,543 |

> **结论**：99.7% 的异常记录中，城市代码与省份代码的省级归属是一致的。

### 2.4 异常城市代码有效性

| 检查项 | 种类数 | 占比 |
|--------|--------|------|
| 异常城市代码总种类 | 1,129 | 100% |
| 是有效区县代码（在字典中存在） | 881 | **78.0%** |
| 无效代码（字典中不存在） | 248 | 22.0% |

无效代码示例：`510124`, `440386`, `440392`, `440358` 等
> 这些可能是历史行政区划变更后的废弃代码，或数据录入错误。

## 五、实现方案

### 5.1 核心修复函数

```python
def derive_correct_city_code(record, city_mapping, district_mapping):
    """
    推导正确的城市代码
    
    :param record: 数据库记录
    :param city_mapping: 城市代码字典 {code: name}
    :param district_mapping: 区县代码字典 {code: name}
    :return: (correct_city_code, strategy_used, confidence)
    """
    current_city = str(record.get('城市', '')).strip()
    district_code = str(record.get('区县', '')).strip()
    province_code = str(record.get('省份', '')).strip()
    address = str(record.get('充电站位置', '')).strip()
    
    # 检查是否需要修复
    if not current_city or current_city.endswith('00'):
        return current_city, 'NO_FIX_NEEDED', 1.0
    
    # ========== 直辖市特殊处理 ==========
    DIRECT_CITY_CODES = {
        '11': '110100', '31': '310100', 
        '12': '120100', '50': '500100'
    }
    
    province_prefix = province_code[:2] if len(province_code) >= 2 else current_city[:2]
    if province_prefix in DIRECT_CITY_CODES:
        return DIRECT_CITY_CODES[province_prefix], 'DIRECT_CITY', 1.0
    
    # ========== 策略1：从区县代码推导 ==========
    if district_code and len(district_code) >= 4:
        derived = district_code[:4] + '00'
        if derived in city_mapping:
            # 校验省份一致性
            if not province_code or derived[:2] == province_code[:2]:
                return derived, 'FROM_DISTRICT', 0.95
    
    # ========== 策略2：从当前城市代码推导 ==========
    if len(current_city) >= 4:
        derived = current_city[:4] + '00'
        if derived in city_mapping:
            # 校验省份一致性
            if not province_code or derived[:2] == province_code[:2]:
                return derived, 'FROM_CURRENT', 0.85
    
    # ========== 策略3：地址解析 ==========
    if address:
        parsed_city = parse_city_from_address(address)
        if parsed_city and parsed_city in city_mapping.values():
            # 反向查找城市代码
            for code, name in city_mapping.items():
                if name == parsed_city:
                    return code, 'FROM_ADDRESS', 0.70
    
    # ========== 无法自动修复 ==========
    return None, 'MANUAL_REQUIRED', 0.0
```

### 5.2 批量修复流程

```python
def batch_fix_city_codes(engine, dry_run=True, batch_size=500):
    """
    批量修复城市代码
    
    Args:
        engine: 数据库引擎
        dry_run: 预览模式（不实际修改）
        batch_size: 每批处理数量
    """
    # 1. 加载映射字典
    city_mapping = load_city_mapping()
    district_mapping = load_district_mapping()
    
    # 2. 查询所有异常记录
    abnormal_records = query_abnormal_city_codes(engine)
    
    # 3. 分析并生成修复方案
    fix_plan = []
    errors = []
    
    for record in abnormal_records:
        correct_code, strategy, confidence = derive_correct_city_code(
            record, city_mapping, district_mapping
        )
        
        if correct_code and confidence >= 0.7:
            fix_plan.append({
                'uid': record['UID'],
                'old_city_code': record['城市'],
                'new_city_code': correct_code,
                'new_city_name': city_mapping.get(correct_code, ''),
                'strategy': strategy,
                'confidence': confidence
            })
        else:
            errors.append({
                'uid': record['UID'],
                'reason': strategy,
                'record': record
            })
    
    # 4. 输出统计
    print(f"待修复记录数: {len(fix_plan)}")
    print(f"无法自动修复: {len(errors)}")
    
    # 5. 执行修复（非预览模式）
    if not dry_run:
        execute_fix(engine, fix_plan, batch_size)
    
    return fix_plan, errors
```

### 6.2 回滚方案

在执行修复前，记录原始值到备份表：

```sql
-- 创建备份表
CREATE TABLE IF NOT EXISTS evdata_city_fix_backup (
    uid VARCHAR(50) PRIMARY KEY,
    old_city_code VARCHAR(10),
    old_city_name VARCHAR(50),
    new_city_code VARCHAR(10),
    new_city_name VARCHAR(50),
    strategy VARCHAR(20),
    confidence DECIMAL(3,2),
    fixed_at DATETIME,
    fixed_by VARCHAR(50)
);

-- 执行修复前插入备份
INSERT INTO evdata_city_fix_backup 
    (uid, old_city_code, old_city_name, fixed_at, fixed_by)
SELECT UID, `城市`, `城市_中文`, NOW(), 'system'
FROM evdata 
WHERE is_active = 1 
AND `城市` IS NOT NULL 
AND RIGHT(CAST(`城市` AS CHAR), 2) != '00';
```

回滚命令：

```sql
-- 从备份恢复
UPDATE evdata e
INNER JOIN evdata_city_fix_backup b ON e.UID = b.uid
SET 
    e.`城市` = b.old_city_code,
    e.`城市_中文` = b.old_city_name
WHERE e.is_active = 1;
```
