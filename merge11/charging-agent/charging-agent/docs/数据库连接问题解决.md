# 数据库连接问题解决方案

## 问题：缺少cryptography包

### 错误信息

```
数据库连接失败: 'cryptography' package is required for sha256_password or caching_sha2_password auth methods
```

### 原因

MySQL 8.0+ 默认使用新的认证方法（`caching_sha2_password` 或 `sha256_password`），这些方法需要 `cryptography` 包来支持加密连接。

### 解决方案

#### 方案1：安装cryptography包（推荐）

**开发环境**：
```bash
pip install cryptography
```

或安装完整依赖：
```bash
pip install -r requirements.txt
```

**EXE打包环境**：
- `cryptography` 已添加到 `requirements.txt`
- 重新打包EXE时会自动包含

#### 方案2：修改MySQL用户认证方法（如果无法安装cryptography）

如果无法安装 `cryptography` 包，可以修改MySQL用户的认证方法为旧的 `mysql_native_password`：

```sql
-- 登录MySQL
mysql -u root -p

-- 修改用户认证方法
ALTER USER 'your_user'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_password';

-- 刷新权限
FLUSH PRIVILEGES;
```

**注意**：此方法会降低安全性，不推荐用于生产环境。

### 验证

安装后，测试数据库连接：

```python
from utils.db_utils import test_connection
success, message = test_connection()
print(message)
```
