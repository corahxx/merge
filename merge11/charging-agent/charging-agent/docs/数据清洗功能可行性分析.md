## 二、功能设计

### 2.1 重复数据检测模块

#### 2.1.1 重复判定规则

| 判定级别 | 条件 | 说明 | 处理建议 |
|----------|------|------|----------|
| **完全重复** | 除UID外所有字段完全相同 | 同一条数据被重复导入多次 | 标记为deleted（软删除） |
| **高度疑似** | 充电桩编号相同 + 地址相同 + 运营商相同 + 功率相同 | 同一个桩被重复录入（编号一致） | 人工核实后标记 |
| **轻度疑似** | 地址相同但运营商不同 | 同一站点多运营商（通常正常） | 仅提示，一般不处理 |

**判定逻辑说明：**

```
完全重复（最高优先级）
├── 条件：除UID（系统自动生成）外，所有业务字段值完全一致
├── 原因：数据重复导入
└── 处理：标记为 deleted（软删除），正常查询自动过滤

高度疑似（中等优先级）
├── 条件：充电桩编号相同 AND 地址相同 AND 运营商相同 AND 功率相同
├── 原因：可能是同一个桩，编号重复录入
└── 处理：人工核实，确认后标记

轻度疑似（低优先级）
├── 条件：地址相同 BUT 运营商不同
├── 原因：同一充电站内有多个运营商的充电桩（正常业务场景）
└── 处理：仅作为提示，通常不需要处理
```

#### 2.1.2 检测SQL示例

```sql
-- 1. 完全重复检测（除UID外所有字段相同）
-- 使用关键业务字段组合判断
SELECT 
    `充电桩编号`,
    `运营商名称`,
    `省份_中文`,
    `城市_中文`,
    `区县_中文`,
    `充电站位置`,
    `充电桩类型_转换`,
    `额定功率`,
    `所属充电站编号`,
    COUNT(*) as 重复次数,
    GROUP_CONCAT(UID SEPARATOR ', ') as UID列表
FROM evdata
GROUP BY 
    `充电桩编号`,
    `运营商名称`,
    `省份_中文`,
    `城市_中文`,
    `区县_中文`,
    `充电站位置`,
    `充电桩类型_转换`,
    `额定功率`,
    `所属充电站编号`
HAVING COUNT(*) > 1
ORDER BY 重复次数 DESC;

-- 2. 高度疑似重复（充电桩编号相同 + 地址 + 运营商 + 功率相同）
SELECT 
    `充电桩编号`,
    `运营商名称`,
    `充电站位置`,
    `额定功率`,
    COUNT(*) as 记录数,
    GROUP_CONCAT(UID SEPARATOR ', ') as UID列表
FROM evdata
GROUP BY `充电桩编号`, `运营商名称`, `充电站位置`, `额定功率`
HAVING COUNT(*) > 1
ORDER BY 记录数 DESC;

-- 3. 轻度疑似（地址相同，运营商不同）
SELECT 
    `充电站位置`,
    COUNT(DISTINCT `运营商名称`) as 运营商数量,
    GROUP_CONCAT(DISTINCT `运营商名称` SEPARATOR ', ') as 运营商列表,
    COUNT(*) as 总记录数
FROM evdata
GROUP BY `充电站位置`
HAVING COUNT(DISTINCT `运营商名称`) > 1
ORDER BY 运营商数量 DESC;
```

#### 2.1.3 标记方案（软删除）

在数据库中新增标记字段：

```sql
-- 活跃标记（用于高效过滤，解决查询性能问题）
ALTER TABLE evdata ADD COLUMN `is_active` TINYINT(1) DEFAULT 1 
    COMMENT '是否活跃: 1=正常显示, 0=已删除不显示';

-- 数据状态（用于记录详细状态）
ALTER TABLE evdata ADD COLUMN `data_status` VARCHAR(20) DEFAULT 'normal' 
    COMMENT '数据状态: normal正常, duplicate重复, suspect疑似, fixed已修复, deleted已删除';

ALTER TABLE evdata ADD COLUMN `duplicate_group_id` INT DEFAULT NULL 
    COMMENT '重复组ID，同组记录值相同';

ALTER TABLE evdata ADD COLUMN `status_note` VARCHAR(255) DEFAULT NULL 
    COMMENT '状态备注';

-- 为is_active创建索引（高效过滤）
CREATE INDEX idx_is_active ON evdata(`is_active`);
```

**双字段设计说明：**

| 字段 | 用途 | 说明 |
|------|------|------|
| `is_active` | **查询过滤** | 简单的0/1值，高效利用索引 |
| `data_status` | **状态记录** | 记录详细状态，用于审计和管理 |

**状态说明：**

| 状态 | is_active | 正常查询 | 说明 |
|------|:---------:|:--------:|------|
| `normal` | 1 | ✅ 显示 | 默认状态 |
| `duplicate` | 1 | ✅ 显示 | 已确认重复，但未删除 |
| `suspect` | 1 | ✅ 显示 | 待进一步核实 |
| `fixed` | 1 | ✅ 显示 | 问题已修正 |
| `deleted` | **0** | ❌ 不显示 | **软删除**，数据保留但不参与查询 |

**软删除策略：**
- 完全重复的数据标记为 `deleted` 且 `is_active=0`，**不物理删除**
- 所有正常查询只需添加 `WHERE is_active = 1`，**高效利用索引**
- 数据仍保留在数据库中，可追溯、可恢复

#### 2.1.4 查询过滤（性能优化）

> **性能优化说明：**
> 
> 使用 `is_active = 1` 代替 `data_status != 'deleted' OR data_status IS NULL`
> - ❌ `data_status != 'deleted' OR data_status IS NULL` → OR条件导致索引失效
> - ✅ `is_active = 1` → 简单等值查询，高效利用索引

所有业务查询使用 `is_active` 过滤：

```sql
-- 正常查询（高效过滤，利用索引）
SELECT * FROM evdata 
WHERE is_active = 1;

-- 统计分析（高效过滤）
SELECT `运营商名称`, COUNT(*) as 数量
FROM evdata 
WHERE is_active = 1
GROUP BY `运营商名称`;

-- 管理员查看所有数据（包括已删除）
SELECT * FROM evdata;  -- 不加过滤条件

-- 仅查看已删除数据（用于恢复或审计）
SELECT * FROM evdata WHERE is_active = 0;
-- 或查看详细状态
SELECT * FROM evdata WHERE data_status = 'deleted';
```

**查询效率对比：**

| 方案 | 查询条件 | 索引利用 | 预估性能 |
|------|----------|:--------:|:--------:|
| 旧方案 | `data_status != 'deleted' OR data_status IS NULL` | ❌ 差 | 慢 |
| 新方案 | `is_active = 1` | ✅ 优秀 | 快 |

> 在百万级数据量下，新方案查询速度预计提升 **3-10倍**

### 2.2 地址一致性校验模块

#### 2.2.1 校验逻辑

```
场站地址（充电站位置）
    ↓ 解析
提取省份、城市、区县
    ↓ 对比
与现有字段比较
    ↓ 不一致时
标记问题，等待人工确认修正
```

#### 2.2.2 地址解析规则

| 场景 | 示例地址 | 解析结果 |
|------|----------|----------|
| 标准格式 | 北京市朝阳区xxx路 | 省=北京市, 市=北京市, 区=朝阳区 |
| 带省份 | 河北省石家庄市长安区xxx | 省=河北省, 市=石家庄市, 区=长安区 |
| 缺少层级 | 朝阳区xxx路 | 需结合现有字段推断 |

#### 2.2.3 不一致检测SQL

```sql
-- 检测省份不一致的记录
SELECT 
    id,
    `充电桩编号`,
    `省份_中文` as 当前省份,
    `城市_中文` as 当前城市,
    `区县_中文` as 当前区县,
    `充电站位置` as 场站地址,
    `运营商名称`
FROM evdata
WHERE 
    -- 地址中不包含当前省份
    `充电站位置` NOT LIKE CONCAT('%', REPLACE(`省份_中文`, '省', ''), '%')
    OR
    -- 地址中不包含当前城市
    `充电站位置` NOT LIKE CONCAT('%', REPLACE(`城市_中文`, '市', ''), '%')
LIMIT 100;
```

### 3.1 新增数据库字段

```sql
-- 活跃标记（用于高效过滤，核心字段）
ALTER TABLE evdata

COMMENT '是否活跃: 1=正常显示, 0=已删除不显示';

-- 数据状态标记（包含软删除状态）
ALTER TABLE evdata 
ADD COLUMN `data_status` ENUM('normal','duplicate','suspect','fixed','deleted') 
DEFAULT 'normal' COMMENT '数据状态: normal正常,duplicate重复,suspect疑似,fixed已修复,deleted已删除(软删除)';

-- 重复组标识
ALTER TABLE evdata

COMMENT '重复组ID';

-- 状态备注
ALTER TABLE evdata 
ADD COLUMN `status_note` VARCHAR(255) DEFAULT NULL 
COMMENT '状态说明';

-- 清洗时间
ALTER TABLE evdata 
ADD COLUMN `cleaned_at` DATETIME DEFAULT NULL 
COMMENT '清洗时间';

-- 清洗人
ALTER TABLE evdata 
ADD COLUMN `cleaned_by` VARCHAR(50) DEFAULT NULL 
COMMENT '清洗操作人';

-- 添加索引（is_active索引最重要，用于高效过滤）
CREATE INDEX idx_is_active ON evdata(`is_active`);
CREATE INDEX idx_data_status ON evdata(`data_status`);
CREATE INDEX idx_duplicate_group ON evdata(`duplicate_group_id`);

-- 初始化：将所有现有数据设为活跃状态
```

### 3.4 地址解析算法

```python
import re

class AddressParser:
    """地址解析器"""
    
    # 直辖市
    DIRECT_CITIES = ['北京', '上海', '天津', '重庆']
    
    # 省份模式
    PROVINCE_PATTERN = r'([\u4e00-\u9fa5]{2,}(?:省|自治区|特别行政区))'
    
    # 城市模式
    CITY_PATTERN = r'([\u4e00-\u9fa5]{2,}(?:市|自治州|地区|盟))'
    
    # 区县模式
    DISTRICT_PATTERN = r'([\u4e00-\u9fa5]{2,}(?:区|县|市|旗))'
    
    def parse(self, address: str) -> Dict:
        """
        解析地址，提取省份、城市、区县
        
        :param address: 原始地址字符串
        :return: {'province': '', 'city': '', 'district': '', 'confidence': 0.0}
        """
        result = {
            'province': None,
            'city': None,
            'district': None,
            'confidence': 0.0
        }
        
        if not address:
            return result
        
        # 1. 检查直辖市
        for city in self.DIRECT_CITIES:
            if city in address:
                result['province'] = f"{city}市"
                result['city'] = f"{city}市"
                # 继续提取区
                district_match = re.search(self.DISTRICT_PATTERN, address)
                if district_match:
                    result['district'] = district_match.group(1)
                result['confidence'] = 0.9
                return result
        
        # 2. 提取省份
        province_match = re.search(self.PROVINCE_PATTERN, address)
        if province_match:
            result['province'] = province_match.group(1)
            result['confidence'] += 0.3
        
        # 3. 提取城市
        city_match = re.search(self.CITY_PATTERN, address)
        if city_match:
            result['city'] = city_match.group(1)
            result['confidence'] += 0.3
        
        # 4. 提取区县
        district_match = re.search(self.DISTRICT_PATTERN, address)
        if district_match:
            result['district'] = district_match.group(1)
            result['confidence'] += 0.3
        
        return result
```

```python
import time
import streamlit as st

class CleaningProgress:
    """清洗进度管理器"""
    
    def __init__(self, total: int):
        self.total = total
        self.processed = 0
        self.start_time = time.time()
        
        # Streamlit 组件
        self.progress_bar = st.progress(0)
        self.status_text = st.empty()
        self.metrics_cols = st.columns(4)
    
    def update(self, processed: int):
        """更新进度"""
        self.processed = processed
        progress = processed / self.total if self.total > 0 else 0
        elapsed = time.time() - self.start_time
        
        # 计算速度和预估时间
        speed = processed / elapsed if elapsed > 0 else 0
        remaining = (self.total - processed) / speed if speed > 0 else 0
        
        # 更新进度条
        self.progress_bar.progress(progress)
        
        # 更新统计信息
        with self.metrics_cols[0]:
            st.metric("已处理", f"{processed:,} / {self.total:,}")
        with self.metrics_cols[1]:
            st.metric("完成", f"{progress:.1%}")
        with self.metrics_cols[2]:
            st.metric("已用时", self._format_time(elapsed))
        with self.metrics_cols[3]:
            st.metric("预估剩余", self._format_time(remaining))
        
        # 更新状态文本
        self.status_text.text(f"⚡ 处理速度: {speed:.1f} 条/秒")
    
    def complete(self):
        """完成处理"""
        elapsed = time.time() - self.start_time
        self.progress_bar.progress(1.0)
        self.status_text.success(
            f"✅ 处理完成！共处理 {self.processed:,} 条，用时 {self._format_time(elapsed)}"
        )
    
    @staticmethod
    def _format_time(seconds: float) -> str:
        """格式化时间"""
        minutes, secs = divmod(int(seconds), 60)
        hours, minutes = divmod(minutes, 60)
        if hours > 0:
            return f"{hours:02d}:{minutes:02d}:{secs:02d}"
        return f"{minutes:02d}:{secs:02d}"
```

```python
def scan_with_progress(handler, mode: str):
    """带进度显示的扫描"""
    
    # 1. 先获取总数
    total = handler.get_total_issues(mode)
    
    if total == 0:
        st.success("✅ 未发现问题数据！")
        return []
    
    # 2. 创建进度管理器
    progress = CleaningProgress(total)
    
    # 3. 分批扫描
    all_issues = []
    batch_size = 100
    
    for offset in range(0, total, batch_size):
        # 获取一批数据
        batch = handler.scan_batch(mode, offset, batch_size)
        all_issues.extend(batch)
        
        # 更新进度
        progress.update(min(offset + batch_size, total))
    
    # 4. 完成
    progress.complete()
    
    return all_issues
```

## 九、缓存与性能优化方案讨论

> **问题背景**：当数据量达到百万级时，如何更科学、高效地进行数据清洗，同时提供更好的用户体验？

### 9.1 方案对比分析

| 方案 | 描述 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **A. 实时查询** | 每次操作直接查询数据库 | 数据实时、简单 | 频繁IO、用户等待 | 数据量小（<10万） |
| **B. 全量缓存** | 一次性加载所有问题数据到内存 | 操作极快 | 内存爆炸、数据可能过期 | 数据量小（<5万） |
| **C. 分批缓存** | 每次加载5-10万条到内存/Session | 平衡内存和性能 | 实现复杂、批次切换慢 | 中等数据量 |
| **D. Redis缓存** | 使用Redis存储问题数据 | 持久化、可共享、高性能 | 需额外部署、增加复杂度 | 大数据量、多用户 |
| **E. 后台任务** | Celery异步扫描+结果存储 | 不阻塞UI、可断点续传 | 架构复杂、需消息队列 | 超大数据量 |

### 9.2 推荐方案：分批缓存 + Session State

**针对当前项目（400万条数据）的推荐方案：**

```
┌────────────────────────────────────────────────────────────────┐
│                     分批缓存架构                                │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   1. 首次扫描                                                  │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐                │
│   │ 数据库   │ -> │ 问题ID   │ -> │ Session  │                │
│   │ 全表扫描 │    │ 列表     │    │ State    │                │
│   └──────────┘    │ (仅存ID) │    │ (内存)   │                │
│                   └──────────┘    └──────────┘                │
│                                                                │
│   2. 分批加载详情                                              │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐                │
│   │ 从ID列表 │ -> │ 查询详情 │ -> │ 当前批次 │                │
│   │ 取100个  │    │ 100条    │    │ 数据     │                │
```

### 9.6 查询条件优化：漏斗式筛选

> **核心思路**：不要一次性比较所有字段，先用简单条件快速缩小范围，再做精细比对。

#### 当前方案（低效）

```sql

FROM evdata
WHERE is_active = 1
```

#### 优化方案（漏斗式筛选）

```

│                    漏斗式筛选流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  第1层：快速筛选（秒级）                                     │
```

```python
class OptimizedDuplicateScanner:
    """漏斗式重复数据扫描器"""
    
    def scan_full_duplicates_optimized(self):
        """优化版完全重复扫描"""
        
        # ===== 第1层：快速找出ID重复的记录 =====
        step1_sql = """
        SELECT `充电桩编号`, COUNT(*) as cnt
        FROM evdata 
        WHERE is_active = 1 AND `充电桩编号` IS NOT NULL
        GROUP BY `充电桩编号`
        HAVING cnt > 1
        """
        duplicate_ids = execute(step1_sql)  # 结果：[('ID001', 3), ('ID002', 2), ...]
        
        if not duplicate_ids:
            return {'total': 0, 'data': []}
        
        # ===== 第2层：只查这些重复ID的详细数据 =====
        id_list = [r[0] for r in duplicate_ids]
        step2_sql = f"""
        SELECT * FROM evdata
        WHERE `充电桩编号` IN ({','.join(['%s']*len(id_list))})
          AND is_active = 1
        ORDER BY `充电桩编号`
        """
        records = execute(step2_sql, id_list)  # 结果：15000条详细记录
        
        # ===== 第3层：内存中精细比对 =====
        df = pd.DataFrame(records)
        
        # 定义比对字段（除UID外的所有关键字段）
        compare_cols = ['充电桩编号', '运营商名称', '省份_中文', '城市_中文',
                        '区县_中文', '充电站位置', '额定功率', '充电桩类型_转换']
        
        # 找出完全重复的组
        full_duplicates = df[df.duplicated(subset=compare_cols, keep=False)]
        
        # 按重复组分组
        groups = full_duplicates.groupby(compare_cols)
        
        result = []
        for _, group in groups:
            if len(group) > 1:  # 确实是重复
                result.append({
                    'uids': group['UID'].tolist(),
                    'count': len(group),
                    'sample': group.iloc[0].to_dict()
                })
        
        return {'total': len(result), 'data': result}
```
