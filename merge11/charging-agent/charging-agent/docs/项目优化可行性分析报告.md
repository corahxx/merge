## 二、优化方向分析

### 2.1 数据库查询速度优化

#### 2.1.1 现状问题

| 问题 | 位置 | 影响 |
|------|------|------|
| **连接池配置缺失** | `utils/db_utils.py` | 每次查询可能创建新连接，增加延迟 |
| **无查询缓存** | `core/query_executor.py` | 重复查询无法复用结果 |
| **全表扫描风险** | `data/data_analyzer.py` | 大数据量统计查询慢 |
| **AI生成SQL无优化** | `core/sql_generator.py` | LLM生成的SQL可能不够优化 |

#### 2.1.2 优化方案

| 方案 | 难度 | 效果 | 优先级 |
|------|------|------|--------|
| **① 连接池优化** | 低 | 高 | P0 |
| **② Streamlit缓存** | 低 | 中高 | P0 |
| **③ 数据库索引建议** | 中 | 高 | P1 |
| **④ 查询结果缓存** | 中 | 中 | P1 |
| **⑤ 分页查询机制** | 中 | 中 | P2 |

**① 连接池优化（推荐立即实施）**
```python
# utils/db_utils.py 修改建议
def create_db_engine(echo: bool = False, **kwargs):
    db_url = get_db_url()
    default_pool_config = {
        'pool_size': 5,           # 连接池大小
        'max_overflow': 10,       # 最大溢出连接
        'pool_pre_ping': True,    # 连接健康检查
        'pool_recycle': 3600,     # 1小时回收
    }
    default_pool_config.update(kwargs)
    return create_engine(db_url, echo=echo, **default_pool_config)
```

**② Streamlit缓存优化**
```python
# 在data_manager.py等文件中使用
@st.cache_data(ttl=300)  # 5分钟缓存
def get_statistics(params):
    ...
```

**③ 数据库索引建议**
```sql
-- 建议添加的索引（需DBA评估）
CREATE INDEX idx_operator ON evdata(运营商名称);
CREATE INDEX idx_region ON evdata(省_中文, 市_中文, 区县_中文);
CREATE INDEX idx_charge_time ON evdata(充电开始时间);
CREATE INDEX idx_station ON evdata(所属充电站编号);
```

#### 2.1.3 预期效果
- 连接池优化：**减少连接建立时间 50-80%**
- Streamlit缓存：**重复查询响应 <100ms**
- 索引优化：**大表查询提速 3-10x**
