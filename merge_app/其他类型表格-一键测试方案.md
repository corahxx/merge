# 其他类型表格合并 — 一键测试功能方案

本文档描述在「**合并汇总其他类型表格**」模块中，在**导出按钮前**新增的「**一键测试**」功能的设计方案，用于通过字段间的加减乘除或大小关系**验证数据的真实性**。**仅作方案说明，不涉及实现细节。**

---

## 一、功能定位

- **位置**：在合并结果展示之后、**导出（下载 Excel/CSV）之前**，增加「一键测试」区块。
- **目的**：对当前合并结果表做**数据真实性校验**——用户配置若干「字段关系规则」，系统按行校验是否成立，并给出通过/不通过及违规行信息。
- **适用**：仅当存在合并结果（即已执行过纵向或横向合并）时显示；无结果时不展示本区块。

---

## 二、规则类型与表达

用户可添加多条「测试规则」，每条规则描述**左侧表达式**与**右侧表达式**的**关系**。

### 2.1 关系运算符

- **算术结果相等**：左侧 = 右侧（允许一定误差，如 1e-6，可配置是否启用容差）。
- **比较关系**：大于（>）、大于等于（≥）、小于（<）、小于等于（≤）、等于（=）、不等于（≠）。

### 2.2 表达式构成

- **左侧 / 右侧**可由以下方式组成（实现时可选支持其中一部分）：
  - **单字段**：从当前结果表中选择一列（如「月度充电电量」）。
  - **字段与常数运算**：如「字段A × 0.5」。
  - **多字段运算**：如「字段A + 字段B」「字段A - 字段B」「字段A × 字段B」「字段A ÷ 字段B」（除法时分母为 0 可约定为不通过或跳过该行）。
  - **常数**：如「100」「0」。
- **建议首期**：左侧 = 字段1 运算符 字段2（或 字段 运算符 常数），右侧 = 字段 或 常数；关系为 =、>、<、≥、≤、≠。  
  例如：「直流桩电量 + 交流桩电量 = 月度充电电量」「汇总 > 0」「字段A ≤ 字段B」。

### 2.3 规则示例

| 说明           | 左侧           | 关系 | 右侧             |
|----------------|----------------|------|------------------|
| 两列之和等于第三列 | 直流桩电量 + 交流桩电量 | =    | 月度充电电量     |
| 汇总大于 0      | 汇总           | >    | 0                |
| A 不超过 B      | 字段A          | ≤    | 字段B            |
| 比例不超过 1    | 占比           | ≤    | 1                |

## 三、交互与 UI 设计

### 3.1 入口与布局

- 在合并结果区（成功提示、指标、预览表）下方、**「📥 导出」标题及导出文件名输入框、下载按钮**之上，增加标题「**🔍 一键测试**」。
- 区块内包含：
  1. **规则列表**：已添加的测试规则（可展示为表格或列表，支持删除、可选项：启用/禁用）。
  2. **添加规则**：通过表单或弹窗新增一条规则（见下）。
  3. **执行测试**按钮：「**一键测试**」。
  4. **测试结果**：执行后展示通过/不通过、每条规则的统计、以及（可选）违规行明细。

### 3.2 添加单条规则

- **左侧表达式**：
  - 方式 A：第一个字段下拉框 + 运算符下拉框（+、-、×、÷）+ 第二个字段或常数输入（若选「常数」则显示数字输入框）。
  - 方式 B：自由输入表达式（需约定列名引用方式，如 `[列名]`），实现复杂度较高，可作后续扩展。
- **关系**：下拉框选择 =、>、≥、<、≤、≠。
- **右侧表达式**：字段下拉框 或 常数输入。
- **容差**（可选）：仅当关系为「=」时显示，如「数值容差 0.000001」，用于浮点比较。

### 3.3 执行与结果展示

- 点击「**一键测试**」后，对当前结果表逐行、逐条规则校验：
  - 若某行某条规则不成立，则该行记为该规则的**违规行**。
- 结果展示建议：
  - **汇总**：共 N 条规则，通过 M 条，不通过 K 条；共 N 行，违规 X 行。
  - **每条规则**：规则描述、是否通过、违规行数；可展开查看「违规行明细」（表格，列出行号或关键列 + 该行相关字段取值）。
  - 若存在违规行，可提供「**导出违规行**」链接或按钮，导出为 Excel/CSV，便于排查。

## 四、计算与异常约定

- **类型**：参与运算的列若非数值，则先尝试转为数值（如 `pd.to_numeric(..., errors="coerce")`），转换失败或为空则该行该规则可视为「跳过」或「不通过」（由产品约定）。
- **空值**：若表达式中某字段在该行为空/NaN，该行该规则可视为「不通过」或「跳过」并在结果中注明。
- **除法**：分母为 0 或空，该行该规则视为「不通过」或「跳过」。
- **比较**：仅当左右两侧均可转为数值时进行大小比较；否则该行该规则不通过或跳过。

---

## 五、与导出的关系

- 一键测试**不改变**合并结果表数据，也不改变导出内容。
- 测试结果（通过/不通过、违规行）**仅用于展示与导出违规行**，不写入主结果表的导出文件（除非用户选择「导出违规行」）。

---

## 六、实施顺序建议

1. 确定表达式能力范围（首期：左 = 字段 运算符 字段/常数，右 = 字段/常数；关系 =、>、<、≥、≤、≠）。
2. 实现「添加规则」表单与规则列表（增删、启用/禁用）。
3. 实现按行、按规则的计算与比较逻辑（含空值、除零、类型转换）。
4. 实现测试结果展示（汇总 + 每规则 + 违规行明细）。
5. 可选：导出违规行、容差配置、更复杂表达式（多字段运算）作为后续迭代。
