# 众链充电原始表合并系统 — 需求汇总（至独立网页、清洗前）

本文档汇总从零在 charging 系统中搭建多表合并功能，到独立为「众链充电原始表合并系统」网页、且在**未加入数据清洗功能之前**的所有需求。便于按此文档从零重新搭建或验收。

---

## 一、系统定位与入口

### 1.1 最终形态（独立网页）

- **系统名称**：众链充电原始表合并系统  
- **运行方式**：独立网页，不连数据库、不需登录；通过 `merge/launcher.py` 或打包后的 exe 一键启动。  
- **入口**：左侧双功能区 —「公共桩多表合并」「充电站多表合并」，右侧主区随选择切换。  
- **与主系统关系**：合并逻辑可在主系统（data_manager）中保留一份；独立 merge 网页为单独入口，仅做合并+预览+导出，不做清洗、不入库。

### 1.2 合并行为总则

- 支持格式：Excel（.xlsx / .xls）、CSV。  
- 多文件上传后：**优先合并能合并的**，不能合并的文件单独列出并写明原因，不因个别文件失败而整批失败。  
- 输出：合并成功数、未合并文件及原因、各表行数（多 sheet 时仅统计主表行数）、合并表行数、合并结果预览（前 10 条）、导出 Excel/CSV。  
- 导出后界面**不重置**，仍保留合并结果与导出按钮，便于再次导出或查看。

## 二、公共桩多表合并

### 2.1 标准列与同义映射

- **标准列（51 列）**：序号、充电桩编号、充电桩内部编号、省份、城市、区县、经度、纬度、经纬度标准、充电桩类型、充电桩所属区域分类、所属充电站编号、充电站内部编号、充电站名称、充电站位置、充电站投入使用时间、充电站所处道路属性、充电站联系电话、充电桩所属运营商、电表号、充电桩厂商编号、充电桩型号、充电桩属性、充电桩生产日期、服务时间、桩型号是否获得联盟标识授权、支付方式、设备开通时间、额定电压上限、额定电压下限、额定电流上限、额定电流下限、额定功率、接口数量、接口1标准～接口4标准、备注、省份_中文、城市_中文、区县_中文、充电桩类型_转换、充电桩属性_转换、充电桩所属运营商_转换、充电桩厂商编号_转换、入库时间、运营商名称、充电桩内部编号_运营商名称。  
- **同义列名映射**（读表后列对齐用）：  
  - 充电桩编码 → 充电桩编号  
  - 充电桩内部编码 → 充电桩内部编号  
  - 充电站编码 → 所属充电站编号  
  - 充电站内部编码 → 充电站内部编号  
  - 运营商机构代码 → 充电桩所属运营商  
  - 接口标准 → 接口1标准  
- 缺列在输出中补空（NaN），列顺序按标准列顺序；不得减少标准列。

### 2.2 单 Sheet 表头规则

- **适用范围**：仅有一个 Sheet 的文件，或多 Sheet 但只使用其中一个 Sheet 时，在该 Sheet 上判定表头。  
- **规则**（行号 0-based，只看前 3 行）：  
  1. 若**第一行（行 0）**包含「单位」「参考」「编码方法」**任一个**：则看**第二行（行 1）**是否包含「充电桩编号」或**同义**（见下）；若包含，则**表头为第二行**，且**第一行不作为数据**，不合并到最终表。  
  2. 若第一行**不**包含上述三个词，且第一行包含「充电桩编号」或同义，则**表头为第一行**。  
  3. 否则：在前 3 行内找**第一个**包含「充电桩编号」或同义的行作为表头；若 3 行内都没有，则报错：**「{表名称}」由于无法确定表头暂未合并**（表名称用文件名或「文件名（Sheet 名）」）。  
- **表头阶段「编号」与「编码」等同**：表头检测时，一行只要包含「充电桩编号」或「充电桩编码」（及同类「XX编码」可视为「XX编号」）即视为满足「充电桩编号」条件。读表后的列名仍按同义映射统一为标准列名。  
- **单 Sheet 特殊约定**：若文件**只有一个 Sheet**，则**不**再判断该 Sheet 是否「有内容」，直接按上述表头规则判定；避免因前几行空白或合并单元格导致误判为「未找到可用的工作表」。

### 2.3 多 Sheet 主表与补全

- **主表选择**：  
  - 若存在**名称中含「1.1」**的 Sheet，则以该 Sheet 为**主表**（多 Sheet 模式）。  
  - 若没有 1.1：则取**第一个「有内容」的 Sheet** 为主表。「有内容」定义为：该 Sheet 前 5 行（header=None）非空单元格数 ≥ 3（若单 Sheet 则不适用此条，见上）。  
- **1.2 / 1.3 补全**（仅当主表为含「1.1」的 Sheet 时执行）：  
  - **1.2**：名称中含「1.2」的 Sheet 作为运营商字典。**表头行**：该 Sheet 中**包含「运营商名称」**的那一行为表头，其下为数据行。用主表（1.1）的「充电桩所属运营商」与 1.2 的「运营商编号」匹配，将 1.2 的「运营商名称」「运营商类型」补全到主表；主表**必须**有这两列（无则新增，可空）。  
  - **1.3**：名称中含「1.3」的 Sheet 作为厂商字典。**表头行**：该 Sheet 中**包含「充电桩生产厂商名称」**的那一行为表头，其下为数据行。用主表（1.1）的「充电桩厂商编号」与 1.3 的「充电桩生产厂商编号」匹配，将 1.3 的「充电桩生产厂商名称」「充电桩生产厂商类型」补全到主表；主表**必须**有这两列（无则新增，可空）。  
- 补全后再做列对齐与纵向合并；多 Sheet 时**行数统计仅计主表（1.1）数据行数**。

### 2.4 上报机构列

- 每个文件处理完后，在**最左侧**增加一列「**上报机构**」。  
- 取值：对**表名称**（一般用文件名）做清洗后填入。  
- **清洗规则**：  
  - 去掉前缀「**202512_公共桩_**」  
  - 去掉「**_公共桩**」  
  - 若存在「**附件一：**」，则删除「附件一：」及其后全部内容  
  - 去掉末尾扩展名（.xlsx / .xls / .csv）及多余下划线、点、空格  
- 示例：`202512_公共桩_嘉兴智行_附件一：xxx.xlsx` → 上报机构为「嘉兴智行」。

### 2.5 合并输出与展示

- **返回/展示内容**：  
  - 合并表行数（总行数）  
  - 合并成功数（成功文件数）  
  - 未合并文件数  
  - **各表行数**：表格两列「文件名」「行数」，多 Sheet 时仅统计主表（1.1）行数  
  - **导出**：Excel、CSV 两个下载按钮（合并表）  
  - **未合并的文件及原因**：列表展示，可放在合并结果预览上方  
  - **合并结果预览**：仅展示前 **10 行**，并注明「仅展示前 10 行，共 N 行」  
- **错误与异常**：  
  - 单个文件读表失败（如 openpyxl 的 filters 报错、无法确定表头等）时，将该文件加入未合并列表并写明原因，其余文件继续合并。  
  - 错误详情可放在页面底部折叠区，不阻塞整体结果展示。

## 三、充电站多表合并

### 3.1 标准列与必填字段

- **必填字段**（可增加、不可减少；导出表必须包含这些列，缺则补空）：  
  所属充电站编号、充电站内部编号、充电站名称、充电站位置、充电站投入使用时间、充电站联系电话、区县、充电站所属区域分类、充电站所属运营商、电表号、充电桩厂商编号、充电桩型号、充电桩属性、充电桩生产日期、服务时间、桩型号是否获得联盟标识授权、支付方式、设备开通时间、额定电压/电流/功率（额定电压上限、额定电压下限、额定电流上限、额定电流下限、额定功率）、接口数量及接口1～4标准、备注、省份_中文、城市_中文、区县_中文、充电桩类型_转换、充电桩属性_转换、充电桩所属运营商_转换、充电桩厂商编号_转换、入库时间、运营商名称、充电桩内部编号_运营商名称等；若业务另有「充电站类型、充电站集中度_转换、站点内桩总数、交流/直流桩数量、站点总装机功率」等站级字段，一并纳入必填列并保持可增不可减。  
- **排除**：列名为「(无列名)」的列不纳入标准列，不在合并输出中出现。  
- 同义映射与表头阶段：**「编号」与「编码」可替换**（如「充电站编码」→「所属充电站编号」、「充电站内部编码」→「充电站内部编号」），表头判定与读表后列对齐均按此处理。**不采用**「充电站编码 + 充电站内部编码」同时出现作为表头行判定条件，仅作列名映射。

### 3.2 单 Sheet 表头规则

- 在前 **3 行**内查找**第一个**包含「**所属充电站编号**」（或同义如「充电站编码」）的行，作为表头行；表头行以下为数据行。  
- 若 3 行内都没有，则报错：「{表名称}由于无法确定表头暂未合并」。

### 3.3 多 Sheet 主表与补全

- **主表选择**（按优先级）：  
  1. 若存在名称含「**1.1**」的 Sheet → 以该 Sheet 为主表。  
  2. 若无 1.1，则取名称含「**充电站**」的 Sheet 为主表。  
  3. 若既无 1.1 也无含「充电站」的 Sheet → 报错「**多sheet表无法确定主表**」。  
- **1.2 / 1.3 的识别**：  
  - 名称含「**1.2**」或含「**运营商**」的 Sheet，视为 1.2（运营商字典）。  
  - 名称含「**1.3**」或含「**厂商**」的 Sheet，视为 1.3（厂商字典）。  
- **补全逻辑**（与公共桩一致）：  
  - 1.2：表头行包含「运营商名称」，主表用「运营商编号」与 1.2「运营商编号」匹配，补全「运营商名称」「运营商类型」到主表。  
  - 1.3：表头行包含「充电桩生产厂商名称」，主表用「充电桩厂商编号」与 1.3「充电桩生产厂商编号」匹配，补全「充电桩生产厂商名称」「充电桩生产厂商类型」到主表。  
- 主表必须包含上述补全列（无则新增，可空）；导出表不得缺少这些列。

### 3.4 上报机构与合并输出

- 上报机构列规则与公共桩一致：最左侧一列「上报机构」，由文件名按相同规则清洗得到（可复用同一套前缀/后缀/「附件一：」规则，若充电站文件名有不同前缀可另行约定）。  
- 合并输出与展示形态与公共桩一致：合并表行数、成功数、未合并数、各表行数、导出、未合并原因、预览 10 条。

## 四、独立 merge 网页的 UI 与运行方式

### 4.1 页面结构

- **标题**：页面顶部标题「众链充电原始表合并系统」，可带简单底图/渐变与副标题区。  
- **侧栏**：左侧固定，默认展开；两个选项「公共桩多表合并」「充电站多表合并」（可用 radio 或带样式的选项卡），选中项持久化（如 session_state）。  
- **主区**：根据侧栏选择渲染对应功能；两套 UI 结构一致，仅合并逻辑与 key 前缀（如 pile_ / station_）不同，避免控件 key 冲突。

### 4.2 公共桩/充电站共用结果区顺序

1. 合并表行数、合并成功数、未合并文件数（三指标）  
2. 各表行数表格（有则展示）  
3. 导出 Excel、导出 CSV  
4. 未合并的文件及原因（列表或 expander）  
5. 合并结果预览（仅前 10 条 + 说明「共 N 行」）  
6. 底部：当前模式（公共桩/充电站）的合并规则说明（expander）

### 4.3 技术约束

- **不依赖主系统 handlers 包**：通过 **importlib** 按文件路径加载 `table_merge_handler.py`、`station_merge_handler.py`，避免 `import handlers` 触发 reportlab 等无关依赖。  
- **充电站未实现时**：`station_merge_handler` 可占位返回 `(None, [], ["充电站合并规则配置中，暂不可用"], [])`，页面仍展示相同 UI 结构并提示暂不可用。

### 4.4 启动方式

- **开发环境**：在项目根执行 `python merge/launcher.py`，或 `streamlit run merge/app.py`（需当前目录为项目根）。  
- **一键启动**：通过 PyInstaller 打包 `merge/launcher.py` 为 exe（如 merge-launcher.exe），双击即可启动；可选启动后自动打开浏览器。打包时注意：使用 .spec 文件时不要传 `--specpath`，且使用项目 venv 的 Python 执行 PyInstaller。
