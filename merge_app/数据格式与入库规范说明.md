# 充电数据格式要求与入库规范说明

本文档汇总当前 Merge 多表合并与数据清洗系统中对**数据格式要求**、**入库格式定义**、**清洗规则**及**字段形式**的约定，便于开发与数据对接时统一口径。

---

## 一、数据格式要求（输入）

### 1.1 支持的文件类型

- **充电桩（公共桩）**：Excel（`.xlsx` / `.xls`）、CSV（UTF-8-sig 或 GBK）。
- **充电站**：同上。
- 合并时支持多文件上传，按统一表头识别后纵向合并，首列填充「上报机构」（由文件名清洗得到）。

### 1.2 表头识别规则

**充电桩表**

- 表头行：前若干行内第一个包含「充电桩编号」或「充电桩编码」的行；若第一行含「单位」「参考」「编码方法」则取第二行含「充电桩编号/编码」为表头。
- 多 Sheet：优先使用名称含「1.1」的 Sheet 作为主表；单 Sheet 则用唯一 Sheet。

**充电站表**

- 表头行：前若干行内第一个包含「所属充电站编号」或「充电站编码」的行。
- 多 Sheet：优先名称含「1.1」的 Sheet，否则含「充电站」的 Sheet；否则报错「多sheet表无法确定主表」。

### 1.3 列名同义映射（合并阶段）

系统在合并时会将下列源列名统一为标准列名（缺列补空）：

## 二、入库格式定义（合并后标准列）

以下为**合并完成后**、**清洗前/后**表结构所依据的标准列集合；清洗不删列，仅对部分列做单位换算、截断、标准化与校验。

### 2.1 充电桩表标准列（公共桩）

合并后首列为「上报机构」，其后列为（顺序见 `table_merge_handler.STANDARD_COLUMNS`）：

- 序号、充电桩编号、充电桩内部编号、省份、城市、区县、经度、纬度、经纬度标准  
- 充电桩类型、充电桩所属区域分类、所属充电站编号、充电站内部编号、充电站名称、充电站位置  
- 充电站投入使用时间、充电站所处道路属性、充电站联系电话、充电桩所属运营商、电表号  
- 充电桩厂商编号、充电桩型号、充电桩属性、充电桩生产日期、服务时间、桩型号是否获得联盟标识授权  
- 支付方式、**设备开通时间**、**额定电压上限**、**额定电压下限**、**额定电流上限**、**额定电流下限**、**额定功率**  
- 接口数量、接口1标准、接口2标准、接口3标准、接口4标准、备注  
- 省份_中文、城市_中文、区县_中文、充电桩类型_转换、充电桩属性_转换、充电桩所属运营商_转换  
- 充电桩厂商编号_转换、入库时间、**运营商名称**、**运营商类型**、充电桩内部编号_运营商名称  
- **充电桩生产厂商名称**、**充电桩生产厂商类型**

合并时「运营商名称」根据**表名称（文件名）**按《数据格式与入库规范说明》1.4 节规则提取填充；1.3 表仅补全厂商名称与类型。

### 2.2 充电站表标准列

合并后首列为「上报机构」，其后列为（顺序见 `station_merge_handler.STATION_STANDARD_COLUMNS`）：

- 所属充电站编号、充电站内部编号、充电站名称、充电站位置、充电站投入使用时间  
- 充电站联系电话、区县、充电站所属区域分类、充电站所属运营商、充电站类型  
- 充电站集中度_转换、服务时间、站点内桩总数、交流桩数量、直流桩数量  
- **站点总装机功率**、**交流桩总装机功率**、**直流桩总装机功率**、备注  
- **运营商名称**、**运营商类型**

不含充电桩专有列（如接口数量、接口1标准等）。

## 三、清洗规则与适用表类型

清洗阶段需先选择**数据类型**：**充电站数据** 或 **充电桩数据**，再执行「一键清洗」（该类型全部规则）或「自定义清洗」（勾选规则子集）。规则按固定顺序执行。

### 3.1 规则 ID 与中文名称

| 规则 ID              | 中文名称                         | 充电站 | 充电桩 |
|-----------------------|----------------------------------|--------|--------|
| null_std              | 空值标准化                       | 是     | 是     |
| sequence              | 主键生成（序号）                 | 是     | 是     |
| location              | 充电站位置截断（≤600字）        | 是     | 是     |
| date                  | 日期清洗（yyyy/mm/dd + 结果列）  | 是     | 是     |
| power                 | 功率→kW                          | 是     | 是     |
| voltage               | 电压→V                           | 是     | 是     |
| current               | 电流→A                           | 是     | 是     |
| station_inner_id      | 充电站内部编号缺失校验           | 是     | —      |
| pile_device_type      | 设备类型标准化（交流/直流）      | —      | 是     |
| pile_open_time        | 设备开通时间校验                 | —      | 是     |

### 3.2 规则说明摘要

**通用（全表）**

- **空值标准化**：`null`、`NULL`、`None`、`nan`、`NaN` 等统一为 `""`。
- **主键生成**：舍弃原 ID，生成从 1 递增的「序号」列，置于首列。
- **充电站位置**：若存在该列，长度超过 600 字符时截断为 600（对应 varchar(600)）。
- **运营商名称**：预留映射接口，清洗逻辑中不写死映射表。

**日期**

- 所有列名中含「时间」「日期」「投入使用」「开通」「生产」「入库」的列，统一转为 **yyyy/mm/dd**；缺失「日」则补 1。
- 每列对应新增「**xxx_日期清洗结果**」：1=成功，0=未识别（保留原值）。报告统计成功数、失败数及无法识别的原始格式样例。

**数值单位**

- **功率**（站点总装机功率、交流桩总装机功率、直流桩总装机功率、额定功率）→ **kW**：含 "W" 不含 "kW" 则÷1000；纯数字且>1000 视为 W 除以 1000。
- **电压**（额定电压上限、额定电压下限）→ **V**：含 "kV" 则×1000；380、220 等保留。
- **电流**（额定电流上限、额定电流下限）→ **A**：剔除 "A" 后保留数字；常见值保留。

**充电站专项**

- 功率列按上条统一为 kW。
- 「充电站内部编号」缺失的行记入报告「充电站内部编号缺失」清单，影响站数统计。

**充电桩专项**

- **设备类型**（充电桩类型）：无汉字且含数字 1→「交流」，无汉字且含数字 2→「直流」；其他（如「01交流」「02直流分体」）暂保持原样。
- **设备开通时间**：须早于当前系统时间，晚于当前时间的行记入「设备开通时间异常」清单。
- **地理位置**：预留高德 API 反查及省市区编码映射接口，当前实现中可为空或注释。

### 3.3 清洗报告输出

- **date_clean_success** / **date_clean_fail**：日期清洗成功/失败单元格数。
- **date_unknown_formats**：无法识别的日期格式样例列表。
- **power_w_to_kw_count**：W→kW 换算条数。
- **station_inner_id_missing_rows**：充电站内部编号缺失记录（行号、序号等）。
- **pile_open_time_anomaly_rows**：设备开通时间晚于当前时间的记录（充电桩表）。
- **applied_rules**：本次实际执行的规则列表 `[(rule_id, 中文名), ...]`，用于界面展示「1、xxx 清洗完成」等。

## 四、字段形式与单位约定（入库后）

| 类别     | 字段（示例）           | 入库格式/单位   | 说明 |
|----------|------------------------|-----------------|------|
| 主键     | 序号                   | 整数，从 1 递增 | 清洗生成，替代不可靠原始 ID |
| 空值     | 全表                   | 空字符串 `""`   | 不保留 null/NULL/文本 "null" |
| 日期     | 设备开通时间、投入使用时间等 | yyyy/mm/dd      | 缺失日补 1；并带「xxx_日期清洗结果」0/1 |
| 功率     | 站点总装机功率、交流/直流桩总装机功率、额定功率 | 数值，单位 **kW** | 禁止删单位，按规则换算 |
| 电压     | 额定电压上限、额定电压下限     | 数值，单位 **V**  | kV×1000；常见值保留 |
| 电流     | 额定电流上限、额定电流下限     | 数值，单位 **A**  | 去单位字符留数字 |
| 文本     | 充电站位置             | 最长 600 字符    | 超长截断 |
| 设备类型 | 充电桩类型（桩表）     | 「交流」/「直流」或原样 | 仅当符合无汉字+数字1/2时改写 |
| 上报     | 上报机构               | 字符串           | 合并时由文件名清洗得到，首列 |

## 五、表类型识别（清洗用）

当用户未选择数据类型时，系统按列名自动识别：

- **充电站**：存在「站点总装机功率」，或存在「充电站内部编号」且不存在「充电桩编号」。
- **充电桩**：存在「充电桩编号」或「额定功率」或「设备开通时间」。
- 若两者皆可则默认按充电站处理；建议在界面上明确选择「充电站数据」或「充电桩数据」后再执行清洗。
