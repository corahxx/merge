# 表格合并规则

本文档描述当前系统中「公共桩多表合并」与「充电站多表合并」的完整规则，便于从 0 到 1 再次开发时复现。**后续规则有更新时，请同步修改本文档。**

---

## 一、总则

- **支持格式**：Excel（.xlsx / .xls）、CSV。
- **合并策略**：多文件上传后**优先合并能合并的**，不能合并的文件单独列出并写明原因，不因个别文件失败而整批失败。
- **合并模式**：在导入表格上方可选「**小文件**」或「**大文件**」。小文件：合并后在页面预览并导出 Excel/CSV；大文件：直接合并为 CSV，仅提供「下载合并结果 CSV」按钮，不展示合并结果预览。
- **输出内容与顺序**：合并表行数、合并成功数、未合并文件数 → 各表行数表格 → 导出 Excel / 导出 CSV（小文件）或 下载合并结果 CSV（大文件）→ **未合并的文件及原因**（以表格列示「文件名」「未合并原因」）→ 合并结果预览（小文件时前 10 行，注明「仅展示前 10 行，共 N 行」）。
- **导出后**：界面不重置，仍保留合并结果与导出按钮。

## 二、公共桩多表合并（充电桩）

### 2.1 标准列（导出表列顺序，缺列补空）

```
序号, 充电桩编号, 充电桩内部编号, 省份, 城市, 区县, 经度, 纬度, 经纬度标准,
充电桩类型, 充电桩所属区域分类, 所属充电站编号, 充电站内部编号, 充电站名称, 充电站位置,
充电站投入使用时间, 充电站所处道路属性, 充电站联系电话, 充电桩所属运营商, 电表号,
充电桩厂商编号, 充电桩型号, 充电桩属性, 充电桩生产日期, 服务时间, 桩型号是否获得联盟标识授权,
支付方式, 设备开通时间, 额定电压上限, 额定电压下限, 额定电流上限, 额定电流下限, 额定功率,
接口数量, 接口1标准, 接口2标准, 接口3标准, 接口4标准, 备注,
省份_中文, 城市_中文, 区县_中文, 充电桩类型_转换, 充电桩属性_转换, 充电桩所属运营商_转换,
充电桩厂商编号_转换, 入库时间, 运营商名称, 运营商类型, 充电桩内部编号_运营商名称,
充电桩生产厂商名称, 充电桩生产厂商类型
```

（最左侧还有一列「上报机构」，由程序在合并后插入，不在此标准列列表中。）

### 2.2 同义列名映射（读表后列对齐用）

| 源列名           | 标准列名       |
|------------------|----------------|
| 充电桩编码       | 充电桩编号     |
| 充电桩内部编码   | 充电桩内部编号 |
| 充电站编码       | 所属充电站编号 |
| 充电站内部编码   | 充电站内部编号 |
| 运营商机构代码   | 充电桩所属运营商 |
| 接口标准         | 接口1标准     |

缺列在输出中补空（NaN），列顺序按标准列；不得减少标准列。

### 2.3 单 Sheet 表头规则

- **适用范围**：仅有一个 Sheet 的文件，或多 Sheet 但只使用其中一个 Sheet 时，在该 Sheet 上判定表头。
- **规则**（行号 0-based，只看前 3 行）：
  1. 若**第一行（行 0）**包含「单位」「参考」「编码方法」**任一个**：则看**第二行（行 1）**是否包含「充电桩编号」或「充电桩编码」；若包含，则**表头为第二行**，且**第一行不作为数据**。
  2. 若第一行不包含上述三个词，且第一行包含「充电桩编号」或「充电桩编码」，则**表头为第一行**。
  3. 否则：在前 3 行内找**第一个**包含「充电桩编号」或「充电桩编码」的行作为表头；若 3 行内都没有，则报错：**「{表名称}」由于无法确定表头暂未合并**（表名称用「文件名」或「文件名（Sheet 名）」）。
- **表头阶段「编号」与「编码」等同**：检测时一行只要包含「充电桩编号」或「充电桩编码」即视为满足条件。读表后列名仍按同义映射统一为标准列名。
- **单 Sheet 特殊约定**：若文件**只有一个 Sheet**，则**不再**判断该 Sheet 是否「有内容」，直接按上述表头规则判定。

### 2.4 多 Sheet 主表与 1.3 补全

- **主表选择**：
  - 若存在名称中含「**1.1**」的 Sheet，则以该 Sheet 为**主表**（多 Sheet 模式）。
  - 若没有 1.1：则取**第一个「有内容」的 Sheet** 为主表。「有内容」定义为：该 Sheet 前 5 行（header=None）非空单元格数 ≥ 3。单 Sheet 时不适用此条。
- **1.2（运营商补全）已取消**：不再读取名称含「1.2」的 Sheet，主表「运营商名称」「运营商类型」两列以空值或原表已有值为准。
- **1.3（厂商字典）**：
  - 名称中含「**1.3**」的 Sheet。**表头行**：该 Sheet 中**包含「充电桩生产厂商名称」**的那一行为表头，其下为数据行。
  - 用主表（1.1）的「充电桩厂商编号」与 1.3 的「充电桩生产厂商编号」匹配，将 1.3 的「充电桩生产厂商名称」「充电桩生产厂商类型」补全到主表；主表必须有两列（无则新增，可空）。
- 补全后再做列对齐与纵向合并；多 Sheet 时**行数统计仅计主表（1.1）数据行数**。

### 2.5 上报机构列

- 每个文件处理完后，在**最左侧**增加一列「**上报机构**」。
- **清洗规则**（对表名称/文件名）：
  - 去掉前缀「**202512_公共桩_**」
  - 去掉「**_公共桩**」
  - 若存在「**附件一：**」，则删除「附件一：」及其后全部内容
  - 去掉末尾扩展名（.xlsx / .xls / .csv）及多余下划线、点、空格
- 示例：`202512_公共桩_嘉兴智行_附件一：xxx.xlsx` → 上报机构为「嘉兴智行」。

### 2.6 运营商名称列

- 合并时「**运营商名称**」根据**表名称（文件名）**按规则提取后填充该文件全部数据行；规则表及后续补充说明见《数据格式与入库规范说明》**1.4 运营商名称按表名称提取**，代码实现见 `handlers/operator_name_rules.py`。

## 三、充电站多表合并

### 3.1 标准列（导出表列顺序，缺列补空；不包含充电桩相关列）

充电站表**不混入充电桩指标**（如接口数量、接口1标准、充电桩厂商编号等）。当前标准列为：

```
所属充电站编号, 充电站内部编号, 充电站名称, 充电站位置, 充电站投入使用时间,
充电站联系电话, 区县, 充电站所属区域分类, 充电站所属运营商, 充电站类型,
充电站集中度_转换, 服务时间, 站点内桩总数, 交流桩数量, 直流桩数量,
站点总装机功率, 交流桩总装机功率, 直流桩总装机功率, 备注,
运营商名称, 运营商类型
```

（最左侧还有一列「上报机构」，由程序在合并后插入。）

### 3.2 同义列名映射

| 源列名           | 标准列名       |
|------------------|----------------|
| 充电站编码       | 所属充电站编号 |
| 充电站内部编码   | 充电站内部编号 |
| 运营商           | 运营商名称     |

列名为「(无列名)」的列不纳入标准列，不在合并输出中出现。

### 3.3 单 Sheet 表头规则

- 在前 **3 行**内查找**第一个**包含「**所属充电站编号**」或「**充电站编码**」的行，作为表头行；表头行以下为数据行。
- 若 3 行内都没有，则报错：「{表名称}由于无法确定表头暂未合并」。

### 3.4 多 Sheet 主表（1.2/1.3 补全已取消）

- **主表选择**（按优先级）：
  1. 若存在名称含「**1.1**」的 Sheet → 以该 Sheet 为主表。
  2. 若无 1.1，则取名称含「**充电站**」的 Sheet 为主表。
  3. 若既无 1.1 也无含「充电站」的 Sheet → 报错「**多sheet表无法确定主表**」。
- **1.2 运营商补全已取消**：不再读取名称含「1.2」或「运营商」的 Sheet，主表「运营商名称」「运营商类型」两列以空值或原表已有值为准。充电站标准列不包含充电桩厂商列，不再读取 1.3。

### 3.5 上报机构列与运营商名称列

- **上报机构**：与公共桩同一套规则，最左侧一列，由文件名按相同规则清洗得到（去掉 202512_公共桩_、_公共桩、附件一：及后缀等）。
- **运营商名称**：根据表名称（文件名）按《数据格式与入库规范说明》1.4 节规则提取后填充，与公共桩共用同一规则表及 `handlers/operator_name_rules.py`。

### 3.6 合并输出与展示

- 与公共桩一致：合并表行数、成功数、未合并数、各表行数、导出 Excel/CSV、未合并原因、预览前 10 条。

## 四、错误与异常

- 单个文件读表失败（如 openpyxl 的 filters 报错、无法确定表头等）时，将该文件加入未合并列表并写明原因，其余文件继续合并。
- **未合并文件输出**：结果区以**表格**列示「**文件名**」与「**未合并原因**」，便于逐条核对。
- 错误详情可放在页面底部折叠区，不阻塞整体结果展示。
